Microsoft Security Bulletin MS05-050 - Kritisch
===============================================

Sicherheitsanfälligkeit in DirectShow kann Remotecodeausführung ermöglichen (904706)
------------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 11. Oktober 2005 | Aktualisiert: Dienstag, 13. Dezember 2005

**Version:** 1.3

#### Zusammenfassung

**Zielgruppe dieses Dokuments:** Benutzer von Microsoft Windows

**Auswirkung der Sicherheitsanfälligkeit:** Codeausführung von Remotestandorten aus

**Bewertung des maximalen Schweregrads:** Kritisch

**Empfehlung:** Wir empfehlen Benutzern die sofortige Installation des Updates.

Kurzzusammenfassung
-------------------

<span></span>
**Kurzzusammenfassung:**

Dieses Update behebt eine neue, von einem Privatanwender entdeckte Sicherheitsanfälligkeit. Die Sicherheitsanfälligkeit wird im Abschnitt „Einzelheiten zu dieser Sicherheitsanfälligkeit“ dieses Bulletins dokumentiert.

Wenn ein Benutzer mit administrativen Benutzerberechtigungen angemeldet ist, könnte ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer könnte dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerberechtigungen erstellen. Für Benutzer, deren Konten mit geringeren Benutzerrechten konfiguriert sind, kann dies geringere Auswirkungen haben als für Benutzer, die mit administrativen Benutzerberechtigungen arbeiten.

Wir empfehlen Benutzern die sofortige Installation des Updates.

Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate  
---------------------------------------------------------------------------
  
<span></span>
**Warum hat Microsoft dieses Bulletin am 13. Dezember 2005 aktualisiert?**  
Microsoft hat dieses Bulletin heute aktualisiert, um Benutzer darauf hinzuweisen, dass eine überarbeitete Version des Sicherheitsupdates für Windows 2000 Service Pack 4, Windows XP Service Pack 1 und Windows 2003 (siehe „Betroffene Software“) vorliegt. Weitere Informationen finden Sie unter „Warum hat Microsoft dieses Bulletin am 9. November 2005 aktualisiert?“
  
Die Updates wurden aktualisiert, um Benutzer auf Installationsfehler hinzuweisen, wenn die falsche Version des DirectX-Pakets auf einem System installiert wird. Benutzer, die die richtige Version von DirectX auf der entsprechenden Version von Windows installiert haben, müssen keine weiteren Maßnahmen ergreifen. Benutzer, die WU, SUS, WSUS, MU, SMS verwenden, müssen keine weiteren Maßnahmen ergreifen, da diese Erkennungsmethoden die richtige Version erkennen und anwenden. Benutzer, die möglicherweise das falsche DirectX-Paket manuell installiert haben, sollten ihre Systeme überprüfen und das richtige Update erneut installieren, um sicherzustellen, dass die richtige Version von DirectX aktualisiert wurde.
  
**Warum hat Microsoft dieses Bulletin am 9. November 2005 aktualisiert?**  
Microsoft wurde gemeldet, dass bei der Installation von „Sicherheitsupdate für DirectX 7.0 für Windows 2000 mit Service Pack 4“ auf einem Computer mit DirectX 8 oder DirectX 9 die Installation erfolgreich abschließt, ohne dass der Computer tatsächlich aktualisiert wurde. Microsoft wurde auch gemeldet, dass bei der Installation von „Sicherheitsupdate für DirectX 8.1 für Windows XP mit Service Pack 1“ oder „Sicherheitsupdate für DirectX 8.1 für Windows 2003“ auf einem Computer mit DirectX 9 die Installation erfolgreich abschließt, ohne dass der Computer tatsächlich aktualisiert wurde. Benutzer, die DirectX 8 oder DirectX 9 ausführen, sind weiterhin für das in MS05-050 beschriebene Problem anfällig, bis sie das richtige Paket für ihre DirectX-Version installiert haben. Weitere Informationen finden Sie in den häufig gestellten Fragen unter „Welches sind die bekannten Probleme, die bei der Installation dieses Sicherheitsupdates auftreten können?“
  
**Welches sind die bekannten Probleme, die bei der Installation dieses Sicherheitsupdates auftreten können?**  
[Im Microsoft Knowledge Base-Artikel 904706](http://support.microsoft.com/kb/904706) werden die derzeit bekannten Probleme bei der Installation dieses Sicherheitsupdates beschrieben. In diesem Artikel werden auch Lösungen für diese Probleme empfohlen. Weitere Informationen finden Sie im [Microsoft Knowledge Base-Artikel 904706](http://support.microsoft.com/kb/904706).
  
-   [Microsoft Knowledge Base-Artikel 909596](http://support.microsoft.com/kb/909596): Bei der Installation einer der in Security Bulletin MS05-050 veröffentlichten Sicherheitsupdates für DirectX auf einem Computer mit Windows Server 2003, Windows XP oder Windows 2000 wird der Computer möglicherweise nicht aktualisiert (909596) (MS05-050).  
-   [Microsoft Knowledge Base-Artikel 910415](http://support.microsoft.com/kb/904706): Ein Windows 2000-basierter Computer, auf dem Windows 2000 Multilingual User Interface (MUI) ausgeführt wird, wird möglicherweise nicht aktualisiert, wenn Sie eine der in Security Bulletin MS05-050 (910415) veröffentlichten Sicherheitsupdates für DirectX installieren.

Ja. Selbst wenn Sie das Sicherheitsupdate für DirectX 8.1 installieren konnten, wurden die Dateien für Versionen, die neuer als DirectX 8.1 sind, nicht aktualisiert.

**Auf meinem Computer wird Windows 2000 Service Pack 4, Windows XP Service Pack 1 oder Windows 2003 ausgeführt. Wie kann ich feststellen, ob das richtige Sicherheitsupdate installiert ist?**  
Mit den folgenden Programmen können Sie überprüfen, ob das richtige Sicherheitsupdate installiert ist: MBSA 2.0, Enterprise Update Scanning Tool, SUS, WSUS oder SMS. Darüber hinaus können Sie auf der Windows Update-Website überprüfen, ob das richtige Sicherheitsupdate installiert ist. Weitere Informationen über die Dateiüberprüfung finden Sie im Abschnitt “DirectX (eigenständig, alle Versionen)” unter “Alle Versionen von DirectX 8 und DirectX 9 unter Windows 2000”, “Alle Versionen von DirectX 9 unter Windows XP” oder “Alle Versionen von DirectX 9 unter Windows 2003”.

**Wie kann ich ermitteln, welches Update auf meinem System installiert werden muss?**
  
Die am Anfang dieses Updates aufgelisteten **Downloads** sind in zwei Kategorien aufgeteilt:
  
1.  Betroffene Software  
2.  Betroffene Komponenten
  
Die neben den Betriebssystemen unter „Betroffene Software“ aufgelisteten Downloads beziehen sich auf Systeme, die nicht auf eine aktuellere Version von DirectX aktualisiert wurden. Falls DirectX auf dem System aktualisiert wurde, muss der Benutzer ein Download aus dem Abschnitt „Betroffene Komponenten“ installieren, das der aktuellen Version von DirectX auf dem jeweiligen System entspricht.
  
**Wie kann ich ermitteln, ob auf meinem System eine aktualisierte Version von DirectX ausgeführt wird?**  
DirectX-Updates werden häufig als empfohlene Updates in Windows Update bereitgestellt. Gehen Sie wie folgt vor, um zu ermitteln, ob Sie eine Aktualisierung zu einer neueren Version von DirectX durchgeführt haben: Klicken Sie auf „DirectX“, klicken Sie dann auf **Start** und auf **Ausführen**, und geben Sie
  
**DXDiag.exe** ein. Klicken Sie dann auf **OK**.
  
Das DirectX-Diagnoseprogramm wird gestartet. Schauen Sie sich die Versionsinformationen neben “DirectX-Version:” an. Dort wird die aktuelle Version von DirectX angezeigt, die auf dem System ausgeführt wird.

#### Häufig gestellte Fragen (FAQs) zur Sicherheitsanfälligkeit in DirectShow – CAN-2005-2128:

**Worin genau besteht diese Sicherheitsanfälligkeit?**
Diese Sicherheitsanfälligkeit kann die Codeausführung von Remotestandorten aus ermöglichen. Nutzt ein Angreifer diese Sicherheitsanfälligkeit erfolgreich aus, kann er von einem Remotestandort aus die vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer könnte dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerberechtigungen erstellen.

**Was ist die Ursache dieser Sicherheitsanfälligkeit?**
Ein ungeprüfter Puffer in DirectShow.

**Was ist DirectShow?**
Microsoft DirectShow dient zum Streaming von Medien auf Betriebssystemen von Microsoft Windows. DirectShow wird verwendet, um qualitativ hochwertige Multimedia-Datenströme zu erfassen und wiederzugeben. Falls vorhanden, erkennt und verwendet es automatisch Hardware zur Video- und Audiobeschleunigung, kann jedoch auch auf Systemen ohne Beschleunigungshardware eingesetzt werden. Zudem ist das Programm mit anderen DirectX-Technologien integriert.
Zu den Anwendungstypen, die mit DirectShow erstellt werden können, gehören DVD-Player, Video-Editing-Anwendungen, AVI-zu-ASF-Konvertierungsprogramme, MP3-Player und Anwendungen zur digitalen Videoaufnahme.

**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**
Wenn ein Angreifer diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann er die vollständige Kontrolle über das betroffene System erlangen.

**Wer könnte diese Sicherheitsanfälligkeit ausnutzen?**
Auf einem Windows-Betriebssystem könnte jeder anonyme Benutzer, der eine speziell gestaltete AVI-Datei an das betroffene System übermittelt, diese Sicherheitsanfälligkeit ausnutzen.

Bei einem E-Mail-Angriff könnte ein Angreifer diese Sicherheitsanfälligkeit ausnutzen, indem er eine speziell gestaltete AVI-Datei an den Benutzer sendet und ihn dazu verleitet, die Datei zu öffnen.
Für ein Angriffsszenario im Web muss ein Angreifer eine Website mit einer Webseite einrichten, die zum Ausnutzen dieser Sicherheitsanfälligkeit verwendet wird. Ein Angreifer kann Benutzer zum Besuch einer Website nicht zwingen. Er muss den Benutzer zu einem Besuch dieser Webseite verleiten. Zu diesem Zweck wird der Benutzer normalerweise dazu gebracht, auf einen Link zur Site des Angreifers zu klicken. Es besteht ebenfalls die Möglichkeit, schädlichen Webinhalt mit Hilfe von Bannerwerbungen anzuzeigen oder Webinhalt auf andere Weise an betroffene Systeme zu übermitteln.

**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**
Arbeitsstationen und Terminalserver sind am meisten gefährdet. Bei Servern ist das Risiko größer, wenn Benutzern ohne ausreichende Verwaltungsberechtigungen erlaubt wird, sich bei Servern anzumelden und Programme auszuführen. Es wird jedoch dringend davon abgeraten, diese Erlaubnis zu erteilen.

**Was bewirkt das Update?**
Das Update behebt die Sicherheitsanfälligkeit, indem die Art der Längenüberprüfung einer Nachricht durch DirectShow vor der Übergabe an den zugewiesenen Puffer geändert wird.

**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**
Nein. Microsoft erhielt Informationen über diese Sicherheitsanfälligkeit durch verantwortungsvolle Offenlegung. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich bekannt war.

**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Auch gab es keine Codebeispiele für ein Angriffskonzept.
