Kurzzusammenfassung
-------------------

<span></span>
**Kurzzusammenfassung:**

Dieses Update behebt eine neu entdeckte, öffentlich bekannte Sicherheitsanfälligkeit. Im Windows FTP-Client liegt eine Sicherheitsanfälligkeit vor, die durch die Art seiner Überprüfung von Dateinamen entsteht. Diese Sicherheitsanfälligkeit könnte es einem Angreifer ermöglichen, während einer FTP-Sitzung unerlaubte Änderungen am Speicherort von Dateiübertragungen auf dem Client vorzunehmen.

Die Sicherheitsanfälligkeit wird im Abschnitt „Einzelheiten zu dieser Sicherheitsanfälligkeit“ dieses Bulletins dokumentiert.

Wir empfehlen Benutzern, dieses Sicherheitsupdate zu installieren.

Einzelheiten zu dieser Sicherheitsanfälligkeit
----------------------------------------------

<span></span>
#### Sicherheitsanfälligkeit im FTP-Client CAN-2005-2126:

Eine Sicherheitsanfälligkeit im Windows FTP-Client kann unerlaubte Änderungen ermöglichen. Durch diese Sicherheitsanfälligkeit könnte ein Angreifer den beabsichtigten Zielspeicherort für eine Dateiübertragung ändern, wenn ein Client eine Datei manuell per FTP überträgt. Durch diese Sicherheitsanfälligkeit könnte der Angreifer die Datei an einen beliebigen Speicherort im Dateisystem auf dem betroffenen System schreiben.

#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit im FTP-Client – CAN-2005-2126:

-   Wenn es einem Angreifer gelingt, Benutzer zum Besuch eines FTP-Servers zu verleiten, auf dem Dateien mit speziell gestalteten Dateinamen gehostet werden, kann er jedoch eine Übertragung dieser Dateien nicht erzwingen. Zum Übertragen von Dateien auf das betroffene System sind Benutzereingriffe erforderlich.
-   Wenn die manipulierte FTP-Übertragung versucht, eine Datei auf dem betroffenen System zu überschreiben, wird der Benutzer in einer Warnmeldung aufgefordert, das Überschreiben der Datei zu bestätigen. Die Datei wird nur dann gespeichert, wenn der Benutzer im Rahmen der Warnmeldung das Überschreiben der Datei bestätigt.

#### Problemumgehungen für die Sicherheitsanfälligkeit im FTP-Client – CAN-2005-2126:

Microsoft hat die folgenden Problemumgehungen getestet. Diese Problemumgehungen beheben nicht die zugrunde liegende Sicherheitsanfälligkeit, sondern blockieren nur die bekannten Angriffsmethoden. Wenn die Funktionalität durch eine Problemumgehung verringert wird, so wird diese Einschränkung im folgenden Abschnitt genannt.

-   **Laden Sie keine Dateien von nicht vertrauenswürdigen FTP-Servern herunter**

    Die Gefahr eines Angriffs kann verringert werden, wenn Sie nur Dateien von vertrauenswürdigen Servern herunterladen.

#### Häufig gestellte Fragen (FAQs) zur Sicherheitsanfälligkeit im FTP-Client – CAN-2005-2126:

**Worin genau besteht diese Sicherheitsanfälligkeit?**
Dies ist eine Sicherheitsanfälligkeit, die unerlaubte Änderungen ermöglicht. Durch diese Sicherheitsanfälligkeit könnte ein Angreifer den beabsichtigten Zielspeicherort für eine Dateiübertragung ändern, wenn ein Client eine Datei manuell per FTP überträgt.

**Was ist die Ursache dieser Sicherheitsanfälligkeit?**
Der Windows FTP-Client führt keine einwandfreie Überprüfung der von FTP-Servern empfangenen Dateinamen durch.

**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, könnte Dateien an bestimmten Speicherorten auf dem betroffenen System speichern. Diese Dateien könnten dann weitere Angriffe ermöglichen. Beispielsweise könnte ein Angreifer eine Programmdatei in einem Ordner wie dem Startordner speichern. Die übertragene Datei würde dann bei der nächsten Benutzeranmeldung ausgeführt.

**Wer könnte diese Sicherheitsanfälligkeit ausnutzen?**
Jeder, der einen Benutzer dazu veranlassen kann, einen FTP-Server mit Dateien mit speziell gestalteten Dateinamen zu besuchen und Dateien von dort zu übertragen.

**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**
Ein Angreifer könnte diese Sicherheitsanfälligkeit ausnutzen, indem er eine Datei mit einem speziell gestalteten Namen auf einem FTP-Server hostet. Der Dateiname muss so gestaltet sein, dass die vom FTP-Client bereitgestellte Dateinamenüberprüfung umgangen wird und er einem gültigen Speicherort auf dem Computer des Benutzer zugeordnet wird. Ein Angreifer muss dann einen Benutzer dazu verleiten, diese Datei herunterzuladen.

**Kann die Sicherheitsanfälligkeit automatisch durch den Besuch eines FTP-Servers ausgenutzt werden?**
Nein. Ein Eingriff des Benutzers ist erforderlich, um die Datei über FTP zu übertragen und zu speichern.

**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**
Bei dieser Sicherheitsanfälligkeit ist es erforderlich, dass ein Benutzer eine Verbindung zu einem FTP-Server herstellt und von dort aus Dateien überträgt. Daher sind besonders Systeme durch diese Sicherheitsanfälligkeit gefährdet, auf denen häufig FTP-Übertragungen stattfinden, wie z. B. Arbeitsstationen. Systeme, auf denen in der Regel FTP-Server besucht werden (z. B. die meisten Serversysteme), sind dagegen weniger gefährdet.

**Was bewirkt das Update?**
Das Update behebt die Sicherheitsanfälligkeit, indem die Art der Überprüfung der von FTP-Servern empfangenen Dateinamen durch den Windows FTP-Client geändert wird.

**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**
Ja. Diese Sicherheitsanfälligkeit wurde veröffentlicht. Ihr wurde die Nummer für allgemeine Sicherheitsanfälligkeit CAN-2005-2126 zugewiesen.

**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**
Nein. Microsoft lagen keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Hingegen waren zum Zeitpunkt der Erstveröffentlichung dieses Bulletins schon Codebeispiele für das Angriffskonzept veröffentlicht worden.

**Werden die Benutzer durch die Installation dieses Sicherheitsupdates vor dem veröffentlichten Code geschützt, der versucht, diese Sicherheitsanfälligkeit auszunutzen?**
Ja. Durch dieses Sicherheitsupdate wird die Sicherheitsanfälligkeit behoben, für die Codebeispiele für ein Angriffskonzept veröffentlicht worden sind.
