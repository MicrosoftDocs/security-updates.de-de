Security Bulletin

Microsoft Security Bulletin MS09-015 - Mittel
=============================================

Sicherheitsanfälligkeit durch komplexe Bedrohung in SearchPath kann Erhöhung von Berechtigungen ermöglichen (959426)
--------------------------------------------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 14. April 2009 | Aktualisiert: Mittwoch, 15. April 2009

**Version:** 1.1

### Allgemeine Informationen

#### Kurzzusammenfassung

Dieses Sicherheitsupdate behebt eine öffentlich gemeldete Sicherheitsanfälligkeit in der Windows SearchPath-Funktion, die eine Erhöhung von Berechtigungen ermöglichen kann, wenn ein Benutzer eine speziell gestaltete Datei an einen bestimmten Ort heruntergeladen und anschließend eine Anwendung geöffnet hat, die die Datei unter bestimmten Umständen laden kann.

Dieses Sicherheitsupdate wird für alle unterstützten Editionen von Windows XP, Windows Server 2003, Windows Vista und Windows Server 2008 als Mittel eingestuft. Unter Microsoft Windows 2000 ist dieses Update nur als Tiefenverteidigungsänderung klassifiziert. Ausführliche Informationen finden Sie in dem Abschnitt **Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate**. Weitere Informationen finden Sie im Unterabschnitt **Betroffene und nicht betroffene Software** in diesem Abschnitt.

Das Sicherheitsupdate behebt die Sicherheitsanfälligkeit, indem geändert wird, wie Windows Dateien vom Desktop lädt. Weitere Informationen zu der Sicherheitsanfälligkeit finden Sie im Unterabschnitt „Häufig gestellte Fragen (FAQs)“ für den bestimmten Sicherheitsanfälligkeitseintrag unter dem nächsten Abschnitt **Informationen zu Sicherheitsanfälligkeiten**.

Mit diesem Sicherheitsupdate wird auch die Sicherheitsanfälligkeit behoben, die erstmals in der [Microsoft-Sicherheitsempfehlung 953818](http://www.microsoft.com/germany/technet/sicherheit/empfehlungen/953818.mspx) beschrieben wurde.

**Empfehlung.** Die Mehrheit der Benutzer hat die automatische Aktualisierung aktiviert und muss keine Maßnahmen ergreifen, da dieses Sicherheitsupdate automatisch heruntergeladen und installiert wird. Benutzer, die die automatische Aktualisierung nicht aktiviert haben, müssen auf Updates prüfen und dieses Update manuell installieren. Weitere Informationen zu bestimmten Konfigurationsoptionen bei der automatischen Aktualisierung finden Sie im [Microsoft Knowledge Base-Artikel 294871](http://support.microsoft.com/kb/294871).

Für Administratoren und für Installationen in Unternehmen bzw. für Endbenutzer, die dieses Sicherheitsupdate manuell installieren möchten, empfiehlt Microsoft, das Update mit Updateverwaltungssoftware zu installieren bzw. mithilfe des Diensts [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) auf Updates zu prüfen.

**Bekannte Probleme.** Keine

Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate
---------------------------------------------------------------------------

<span></span>
**Wo befinden sich die Dateiinformationen?**  
Die Dateiinformationen befinden sich im [Microsoft Knowledge Base-Artikel 959426](http://support.microsoft.com/kb/959426).

**In welcher Beziehung steht dieses Sicherheitsupdate zu MS09-014?**  
Die Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung, [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540), die in diesem Security Bulletin beschrieben wird, betrifft auch Internet Explorer. Microsoft empfiehlt Benutzern, auch das in [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), Kumulatives Sicherheitsupdate für Internet Explorer (963027), bereitgestellte Sicherheitsupdate zu installieren.

**Enthält dieses Update sicherheitsbezogene Funktionsänderungen?**  
Ja. Zusätzlich zu den Änderungen, die im Abschnitt **Informationen zu Sicherheitsanfälligkeiten** dieses Bulletins aufgeführt sind, enthält dieses Update eine Tiefenverteidigungsänderung, durch die eine neue API namens „SetSearchPathMode“ eingeführt wird.

**Was ermöglicht die SetSearchPathMode-Tiefenverteidigungsänderung?**  
Diese neue API ermöglicht einen Pro-Prozessmodus bei der Verwendung der SearchPath-Funktion zum Suchen von Dateien. Dies ermöglicht Anwendungen, das aktuelle Verzeichnis zu zwingen, nach der Anwendung und den Systemorten durchsucht zu werden.

**Werde ich vollständig vor dem komplexen Bedrohungsproblem, das in CVE-2008-2540 beschrieben wird, geschützt sein, wenn ich nur dieses Sicherheitsupdate installiere?**  
Nein, Sie müssen außerdem den Tiefenverteidigungsschutz separat aktivieren, der durch das Kumulative Sicherheitsupdate für Internet Explorer, [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), bereitgestellt wird. Sie müssen jedoch überprüfen, ob die Aktivierung dieses Tiefenverteidigungsschutzes gegen das komplexe Bedrohungsproblem für Ihre Umgebung geeignet ist.

Um den Tiefenverteidigungsschutz in Internet Explorer zu aktivieren, aktivieren Sie den SearchPath-Funktionssteuerungsschlüssel, der im Abschnitt „Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate“ in [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659) beschrieben wurde. Weitere Informationen zum komplexen Bedrohungsproblem finden Sie unter der Nummer für allgemeine Sicherheitsanfälligkeit [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).

**Warum ist Microsoft Windows 2000 in der Tabelle „Betroffene Software“ ohne Sicherheitsauswirkung und Bewertung des Schweregrads aufgeführt?**
Microsoft Windows 2000 ist ohne Sicherheitsauswirkung und Bewertung des Schweregrads aufgeführt, weil dieses Update nicht die gleiche Problembehebung für Microsoft Windows 2000 wie für andere unterstützte Windows-Betriebssysteme enthält. Stattdessen enthält dieses Update nur eine Tiefenverteidigungsänderung, die spezifisch für Microsoft Windows 2000 ist.

**Warum unterscheidet sich die Problembehebung für das komplexe Bedrohungsproblem in Microsoft Windows 2000 von anderen Plattformen? **
Die Architektur zum Implementieren der SetSearchPathMode-Funktion ist nicht auf dem Microsoft Windows 2000-System vorhanden, wodurch die Sicherheitsanfälligkeit in Microsoft Windows 2000 nicht beseitigt werden kann. Dies würde einen Neuentwurf der Architektur eines bedeutenden Teils des Microsoft Windows 2000-Betriebssystems und nicht nur der betroffenen Komponente erforderlich machen. Das Ergebnis solcher Bemühungen wäre nur unzureichend mit Microsoft Windows 2000 kompatibel, sodass es keine Garantie dafür gäbe, dass Anwendungen, die für Microsoft Windows 2000 entworfen wurden, weiterhin auf dem neuentworfenen System funktionieren. Microsoft sind jedoch keine gültigen Angriffsmethoden bekannt, die auf diese Sicherheitsanfälligkeit unter Microsoft Windows 2000 abzielen würden.

Aufgrund dieser architektonischen Änderungen von Microsoft Windows 2000 müssen Sie die für Microsoft Windows 2000 spezifische Tiefenverteidigungsänderung mithilfe des Registrierungsschlüssels HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode aktivieren. Anweisungen hierzu finden Sie im [Microsoft Knowledge Base-Artikel 959426](http://support.microsoft.com/kb/959426). Weitere Informationen finden Sie im MSDN-Artikel [DLL-Suchreihenfolge](http://msdn.microsoft.com/en-us/library/ms682586.aspx).

**Was ist die SearchPath-Funktion?**  
Die SearchPath-Funktion wird verwendet, um eine bestimmte Datei in einem angegebenen Pfad zu suchen. Weitere Informationen finden Sie in dem MSDN-Artikel [SearchPath-Funktion](http://msdn.microsoft.com/en-us/library/aa365527.aspx).

**Was ist die LoadLibrary-Funktion?**  
Die LoadLibrary-Funktion lädt das angegebene Modul in den Adressbereich des aufrufenden Prozesses. Das angegebene Modul kann das Laden anderer Module verursachen. Weitere Informationen finden Sie in dem MSDN-Artikel [LoadLibrary-Funktion](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx).

Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung – CVE-2008-2540  
------------------------------------------------------------------------------------------------------
  
<span></span>
Eine Sicherheitsanfälligkeit bezüglich der Erhöhung von Berechtigungen durch eine komplexe Bedrohung wird durch die Art und Weise verursacht, wie die SearchPath-Funktion in Windows Dateien auf dem System sucht und öffnet. Ein Angreifer könnte die Sicherheitsanfälligkeit dadurch ausnutzen, dass er einen Benutzer dazu verleitet, eine speziell gestaltete Datei an einen bestimmten Ort herunterzuladen und dann eine Anwendung zu öffnen, die die Datei unter bestimmten Umständen lädt.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).
  
#### Schadensbegrenzende Faktoren für Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung – CVE-2008-2540
  
Für diese Sicherheitsanfälligkeit gibt es noch keine schadensbegrenzenden Faktoren.
  
#### Problemumgehungen für Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung – CVE-2008-2540

#### Häufig gestellte Fragen zur Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung – CVE-2008-2540
  
**Worin genau besteht diese Sicherheitsanfälligkeit?**    
Dies ist eine Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung. Nutzt ein Angreifer diese Sicherheitsanfälligkeit erfolgreich aus, kann er die vollständige Kontrolle über das betroffene System erlangen. Ein Angreifer kann dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerrechten erstellen.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Internet Explorer kann aufgrund der Position der Anwendungsbibliotheken im Windows-System u. U. eine speziell gestaltete Datei vom Desktop öffnen. Eine komplexe Bedrohung, bei der Dateien ohne Aufforderung auf das System eines Benutzers heruntergeladen werden, kann bewirken, dass Internet Explorer diese heruntergeladene Datei unbeabsichtigt vom Desktop und nicht aus dem Windows-Systems lädt.
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann beliebigen Code ausführen. Ein Angreifer kann dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerrechten erstellen.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
Ein Angreifer könnte eine speziell gestaltete Datei erstellen und dann einen Benutzer dazu veranlassen, die Datei auf den Desktop herunterzuladen. Der Benutzer müsste dann eine Anwendung starten, die diese speziell gestaltete Datei öffnet. Ein Angreifer kann Benutzer jedoch nicht dazu zwingen, solche Dateien herunterzuladen und die Dateien auf dem Desktop abzulegen.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Alle Systeme, auf denen Benutzer Dateien herunterladen und Anwendungen starten, wie z. B. Arbeitsstationen und Terminalserver, sind durch diese Sicherheitsanfälligkeit gefährdet.
  
**Was bewirkt das Update?**    
Das Update ändert die Suchreihenfolge für Anwendungsbibliotheken, sodass das Systemverzeichnis zuerst gesucht wird, bevor die aktuellen Arbeitsverzeichnisse gesucht werden. Das Update stellt auch eine Tiefenverteidigungsänderung in Form einer neuen API namens „SetSearchPathMode“ bereit, um fortgeschrittenen Benutzern zu helfen, die gewünschte Suchreihenfolge für die SearchPath-Funktion festzulegen. Weitere Informationen zu dieser Tiefenverteidigungsänderung finden Sie in dem Abschnitt **Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate**.
  
**Weshalb wurde diese Sicherheitsanfälligkeit und der Sicherheitsanfälligkeit bezüglich Erhöhung von Berechtigungen durch komplexe Bedrohung die gleiche CVE-Nummer (CVE-2008-2540) in MS09-014, Kumulatives Sicherheitsupdate für Internet Explorer (963027), zugewiesen?**  
[CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) betrifft sowohl Internet Explorer 7 für Windows XP Service Pack 2 und Windows XP Service Pack 3, Internet Explorer 7 für Windows XP Professional x64 Edition und Windows XP Professional x64 Edition Service Pack 2 einerseits als auch die SearchPath-Funktion von Windows andererseits. Microsoft empfiehlt Benutzern, auch das in [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), Kumulatives Sicherheitsupdate für Internet Explorer (963027), bereitgestellte Sicherheitsupdate zu installieren. Die Tiefenverteidigungsänderung für diese Sicherheitsanfälligkeit in diesem Update entspricht auch der Problembehebung in [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). Weitere Informationen finden Sie in den Häufig gestellten Fragen „Enthält dieses Update sicherheitsbezogene Funktionsänderungen?“ im Abschnitt **Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate.**
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Ja. Diese Sicherheitsanfälligkeit wurde veröffentlicht. Ihr wurde die Nummer für allgemeine Sicherheitsanfälligkeit [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) zugewiesen. Dieses Sicherheitsupdate behebt auch die Sicherheitsanfälligkeit, die zuerst in der [Microsoft-Sicherheitsempfehlung 953818](http://www.microsoft.com/germany/technet/sicherheit/empfehlungen/953818.mspx) beschrieben wurde.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Auch gab es keine Codebeispiele für ein Angriffskonzept.

    2.  Wenn die Datei unter **Programme** angezeigt wird, klicken Sie mit der rechten Maustaste auf den Dateinamen, und klicken Sie dann auf **Eigenschaften**.  
    3.  Vergleichen Sie in der Registerkarte **Allgemein** die Dateigröße mit den im Knowledge Base-Artikel des Bulletins bereitgestellten Dateiinformationstabellen.  
    4.  Sie können auch auf die Registerkarte **Detail** klicken und die Informationen, z. B. zur Dateiversion und zum Änderungsdatum, mit den vorher im Knowledge Base-Artikel des Bulletins bereitgestellten Dateiinformationstabellen vergleichen.  
    5.  Außerdem können Sie auch auf die Registerkarte **Vorherige Versionen** klicken und die Dateiinformationen zur vorherigen Dateiversion mit den Dateiinformationen für die neue, bzw. aktualisierte, Dateiversion vergleichen.
