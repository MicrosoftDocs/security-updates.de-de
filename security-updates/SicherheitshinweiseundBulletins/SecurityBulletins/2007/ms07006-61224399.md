Security Bulletin

Microsoft Security Bulletin MS07-006 - Hoch
===========================================

Sicherheitsanfälligkeit in Windows Shell kann Erhöhung von Berechtigungen ermöglichen (928255)
----------------------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 13. Februar 2007 | Aktualisiert: Donnerstag, 15. Februar 2007

**Zielgruppe dieses Dokuments:** Benutzer von Microsoft Windows

**Auswirkung der Sicherheitsanfälligkeit:** Erhöhung von Berechtigungen

Einzelheiten zu dieser Sicherheitsanfälligkeit
----------------------------------------------

<span></span>
#### Sicherheitsanfälligkeit in Windows Shell bezüglich Hardwareerkennung – CVE-2007-0211:

In Windows Shell liegt eine Sicherheitsanfälligkeit bezüglich der Erhöhung von Berechtigungen ([Privilege Elevation](http://go.microsoft.com/fwlink/?linkid=21142)) aufgrund der Art vor, in der das Betriebssystem Erkennung und Registrierung neuer Hardware durchführt. Durch diese Sicherheitsanfälligkeit kann ein angemeldeter Angreifer vollständige Kontrolle über das System erlangen.

#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit in Windows Shell bezüglich Hardwareerkennung – CVE-2007-0211:

-   Ein Angreifer benötigt gültige Anmeldeinformationen, um diese Sicherheitsanfälligkeit auszunutzen. Die Sicherheitsanfälligkeit kann nicht von anonymen Benutzern ausgenutzt werden.
-   Unter Windows XP Service Pack 2 und Windows Server 2003 Service Pack 1 müsste ein Angreifer über Administratorrechte verfügen, um die Sicherheitsanfälligkeit von einem Remotestandort aus ausnutzen zu können.

#### Problemumgehungen für die Sicherheitsanfälligkeit in Windows Shell bezüglich Hardwareerkennung – CVE-2007-0211:

Microsoft hat die folgenden Problemumgehungen getestet. Diese Problemumgehungen beheben nicht die zugrunde liegende Sicherheitsanfälligkeit, sondern blockieren nur die bekannten Angriffsmethoden. Wenn die Funktionalität durch eine Problemumgehung verringert wird, so wird diese Einschränkung im folgenden Abschnitt genannt.

-   **Deaktivieren Sie die Shellhardwareerkennung**

    Das Deaktivieren der Shellhardwareerkennung trägt zum Schutz vor Angriffen bei, mit denen diese Sicherheitsanfälligkeit ausgenutzt wird. Führen Sie die folgenden Schritte aus, um die Shellhardwareerkennung zu deaktivieren:

    1.  Klicken Sie auf **Start** und anschließend auf **Systemsteuerung** .Oder zeigen Sie auf **Einstellungen**, und klicken Sie dann auf **Systemsteuerung**.
    2.  Doppelklicken Sie auf **Verwaltung**.
    3.  Doppelklicken Sie auf **Dienste**.
    4.  Doppelklicken Sie auf **Shellhardwareerkennung**.
    5.  Klicken Sie in der Liste **Starttyp** auf **Deaktiviert**.
    6.  Klicken Sie auf **Beenden** und anschließend auf **OK**.

    Sie können die Shellhardwareerkennung auch deaktivieren, indem Sie den folgenden Befehl an der Eingabeaufforderung eingeben:

    `sc stop ShellHWDetection & sc config ShellHWDetection start= disabled`

    **Auswirkung der Problemumgehung:** Wenn Sie die **Shellhardwareerkennung** deaktivieren, können Sie u. U. den schnellen Benutzerwechsel ([Fast User Switching](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/fast_user_switching.mspx)) nicht verwenden. Daher wird diese Problemumgehung nur für Systeme empfohlen, auf denen diese Funktion nicht erforderlich ist.

#### Häufig gestellte Fragen zur Sicherheitsanfälligkeit in Windows Shell bezüglich Hardwareerkennung – CVE-2007-0211:

**Worin genau besteht diese Sicherheitsanfälligkeit?**
Es handelt sich bei dieser Sicherheitsanfälligkeit um eine Erhöhung von Berechtigungen ([Privilege Elevation](http://go.microsoft.com/fwlink/?linkid=21142)). Nutzt ein Angreifer diese Sicherheitsanfälligkeit erfolgreich aus, kann er die vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer könnte dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerrechten erstellen. Zum Ausnutzen dieser Sicherheitsanfälligkeit muss der Angreifer gültige Anmeldeinformationen für das System haben und ein Programm ausführen.

**Was ist die Shellhardwareerkennung?**
Die Shellhardwareerkennung zeigt Benachrichtigungen für Hardwareereignisse der automatischen Wiedergabe an.

**Was ist die Ursache dieser Sicherheitsanfälligkeit?**
Ein Funktionsparameter in der Hardwareinitialisierung und den Erkennungsfunktionen von Microsoft Windows wurde nicht überprüft.

**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**
Wenn ein Angreifer diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann er vollständige Kontrolle über das betroffene System erlangen.

**Wer könnte diese Sicherheitsanfälligkeit ausnutzen?**
Zum Ausnutzen dieser Sicherheitsanfälligkeit muss der Angreifer gültige Anmeldeinformationen für das System haben und ein Programm ausführen.

**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**
Zum Ausnutzen dieser Sicherheitsanfälligkeit muss ein Angreifer sich zuerst beim System anmelden und eine speziell gestaltete Anwendung ausführen. Dann könnte ein Angreifer das betroffene System zwingen, eine Hardwareerkennung und ein Initialisierungsereignis durchzuführen. Die Anforderung könnte lokal oder durch eine Terminaldienstsitzung ausgegeben werden, indem eine speziell gestaltete Anwendung, ein Hardwaregerät oder Speichermedium aufgerufen wird, mit dem die Sicherheitsanfälligkeit ausgenutzt und vollständige Kontrolle über das betroffene System erlangt werden kann.

**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**
Arbeitsstationen und Terminalserver sind am meisten gefährdet. Bei Servern ist das Risiko größer, wenn Administratoren Benutzern ermöglichen, sich bei Servern anzumelden und Programme auszuführen. Es wird jedoch dringend davon abgeraten, diese Erlaubnis zu erteilen.

**Kann diese Sicherheitsanfälligkeit über das Internet ausgenutzt werden?**
Nein. Zum Durchführen eines Angriffs müsste ein Angreifer in der Lage sein, sich am jeweiligen System anzumelden, das angegriffen werden soll. Ein nicht authentifizierter Angreifer kann diese Sicherheitsanfälligkeit nicht ausnutzen, um von einem Remotestandort aus ein Programm zu laden und auszuführen.

**Was bewirkt das Update?**
Das Update behebt die Sicherheitsanfälligkeit, indem der Parameter der betroffenen Funktion überprüft wird.

**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**
Nein. Microsoft erhielt Informationen über diese Sicherheitsanfälligkeit durch verantwortungsvolle Offenlegung. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich bekannt war.

**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Auch gab es keine Codebeispiele für ein Angriffskonzept.

**In welcher Beziehung steht diese Sicherheitsanfälligkeit zu der durch MS06-045 behobenen Sicherheitsanfälligkeit?**
Beide Sicherheitsanfälligkeiten befanden sich in Windows Shell, doch dieses Update behebt eine neue Sicherheitsanfälligkeit, die nicht als Teil von [MS06-045](http://www.microsoft.com/germany/technet/sicherheit/bulletins/ms06-045.mspx) behandelt wurde. MS06-045 bietet Schutz vor der im entsprechenden Bulletin beschriebenen Sicherheitsanfälligkeit, behebt aber nicht diese neue Sicherheitsanfälligkeit. Das vorliegende Update ersetzt MS05-045. Sie können dieses Update installieren, um Ihr System vor beiden Sicherheitsanfälligkeiten zu schützen.
