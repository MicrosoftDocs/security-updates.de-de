Security Bulletin

Microsoft Security Bulletin MS10-030 - Kritisch
===============================================

Sicherheitsanfälligkeit in Outlook Express und Windows Mail kann Remotecodeausführung ermöglichen (978542)
----------------------------------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 11. Mai 2010 | Aktualisiert: Mittwoch, 19. Mai 2010

**Version:** 1.2

### Allgemeine Informationen

#### Kurzzusammenfassung

Dieses Sicherheitsupdate behebt eine vertraulich gemeldete Sicherheitsanfälligkeit in Outlook Express, Windows Mail und Windows Live Mail. Die Sicherheitsanfälligkeit kann Remotecodeausführung ermöglichen, wenn ein Benutzer einen schädlichen E-Mail-Server besucht. Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann die gleichen Benutzerrechte wie der lokale Endbenutzer erlangen. Für Endbenutzer, deren Konten mit weniger Benutzerrechten konfiguriert sind, kann dies geringere Auswirkungen haben als für Benutzer, die mit administrativen Benutzerrechten arbeiten.

Dieses Sicherheitsupdate wird für Microsoft Outlook Express unter allen unterstützten Editionen von Microsofts Windows 2000, Windows XP und Windows Server 2003 als Kritisch eingestuft; ebenso für Windows Mail unter allen unterstützten Editionen von Windows Vista und Windows Server 2008. Dieses Sicherheitsupdate wird für Windows Live Mail unter allen unterstützten Editionen von Windows XP, Windows Vista, Windows Server 2008, Windows 7 und Windows Server 2008 R2 als Hoch eingestuft. Weitere Informationen finden Sie in dem Unterabschnitt **Betroffene und nicht betroffene Software** in diesem Abschnitt.

Das Sicherheitsupdate behebt die Sicherheitsanfälligkeit, indem die Antworten des E-Mail-Servers richtig überprüft werden. Weitere Informationen zur Sicherheitsanfälligkeit finden Sie im Unterabschnitt „Häufig gestellte Fragen (FAQs)“ für den bestimmten Sicherheitsanfälligkeitseintrag unter dem nächsten Abschnitt **Informationen zu Sicherheitsanfälligkeiten**.

**Empfehlung.** Die Mehrheit der Benutzer hat die automatische Aktualisierung aktiviert und muss keine Maßnahmen ergreifen, da dieses Sicherheitsupdate automatisch heruntergeladen und installiert wird. Benutzer, die die automatische Aktualisierung nicht aktiviert haben, müssen auf Updates prüfen und dieses Update manuell installieren. Weitere Informationen zu bestimmten Konfigurationsoptionen bei der automatischen Aktualisierung finden Sie im [Microsoft Knowledge Base-Artikel 294871](http://support.microsoft.com/kb/294871/de).

Für Administratoren und für Installationen in Unternehmen bzw. für Endbenutzer, die dieses Sicherheitsupdate manuell installieren möchten, empfiehlt Microsoft, das Update sofort mit Updateverwaltungssoftware zu installieren bzw. mithilfe des Diensts [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) auf Updates zu prüfen.

Siehe auch den Abschnitt **Tools und Anleitungen zur Erkennung und Bereitstellung** weiter unten in diesem Bulletin.

**Bekannte Probleme.** Keine

Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate  
---------------------------------------------------------------------------
  
**Warum wird diese Sicherheitsanfälligkeit unter Windows 7 und Windows Server 2008 R2 als Hoch eingestuft?**    
Unter Windows 7 und Windows Server 2008 R2 ist Windows Live Mail eine gebrauchsfertige Komponente, die standardmäßig nicht installiert ist. Ein Benutzer muss die Anwendung separat installieren, um durch dieses Problem betroffen zu sein. Der anfällige Code ist jedoch in einer Standardinstallation in der Datei „inetcomm.dll“ auf dem Computer vorhanden, und durch die Installation dieses Updates wird die Sicherheitsanfälligkeit behoben, unabhängig davon, ob die Angriffsmethode existiert.
  
**Ich führe Windows 7 oder Windows Server 2008 R2 aus, ohne Windows Live Mail installiert zu haben. Warum wird mir immer noch dieses Sicherheitsupdate angeboten?**    
Diese Sicherheitsanfälligkeit liegt in der „inetcomm.dll“ vor, die eine Kernkomponente des Betriebssystems ist. Die einzige Angriffsmethode jedoch eröffnet sich durch die Verwendung von Windows Live Mail. Ihnen wird dieses Sicherheitsupdate immer noch als Tiefenverteidigungsupdate angeboten, um die Sicherheitsanfälligkeit auf dem System vollständig zu beheben.
  
**Wo befinden sich die Dateiinformationen?**    
Informationen zum Speicherort der Dateiinformationen finden Sie in den Referenztabellen im Abschnitt **Bereitstellung von Sicherheitsupdates**.

Sicherheitsanfälligkeit in Outlook Express und Windows Mail bezüglich Ganzzahlüberlaufs – CVE-2010-0816  
-------------------------------------------------------------------------------------------------------
  
Es liegt eine Sicherheitsanfälligkeit bezüglich der Ausführung nicht authentifizierter Remotecodes vor, die dadurch verursacht wird, wie der Windows E-Mail-Client speziell gestaltete Mail-Antworten verarbeitet. Ein Versuch, die Sicherheitsanfälligkeit auszunutzen, erfordert keine Authentifizierung. Dadurch kann ein Angreifer die Sicherheitsanfälligkeit ausnutzen, indem er eine speziell gestaltete Antwort an einen Client sendet, mit der mithilfe der gängigen Mail-Protokolle POP3 und IMAP eine Verbindung zu einem Server initiiert wird, der vom Angreifer kontrolliert wird.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2010-0816](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0816).
  
#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit in Outlook Express und Windows Mail bezüglich Ganzzahlüberlaufs – CVE-2010-0816
  
Schadensbegrenzung bezieht sich auf eine Einstellung, häufige Konfiguration oder allgemeine empfohlene Vorgehensweise, die in einem Standardzustand existieren und den Schweregrad der Ausnutzung einer Sicherheitsanfälligkeit verringern können. Die folgenden schadensbegrenzenden Faktoren könnten hilfreich für Sie sein:
  
-   Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann die gleichen Benutzerrechte wie der lokale Endbenutzer erlangen. Für Endbenutzer, deren Konten mit weniger Benutzerrechten konfiguriert sind, kann dies geringere Auswirkungen haben als für Benutzer, die mit administrativen Benutzerrechten arbeiten.  
-   Ein Angreifer kann keine schädliche E-Mail senden, um diese Sicherheitsanfälligkeit auszulösen.  
-   Benutzer, die zusätzlich zu den POP3- oder IMAP-Protokollen SSL-Verschlüsselung verwenden, sind nicht mehr für Man-in-the-Middle-Angriffe anfällig, mit denen diese Sicherheitsanfälligkeit ausgenutzt wird. Sie sind nur betroffen, wenn der Client dazu verleitet werden kann, eine Verbindung zu einem schädlichen oder beeinträchtigten E-Mail-Server herzustellen.
  
#### Problemumgehungen für die Sicherheitsanfälligkeit in Outlook Express und Windows Mail bezüglich Ganzzahlüberlaufs – CVE-2010-0816
  
Problemumgehung bezieht sich auf eine Einstellung oder Konfigurationsänderung, die die zugrunde liegende Sicherheitsanfälligkeit nicht behebt, sondern die bekannten Angriffsmethoden blockiert, bevor Sie das Update installieren. Microsoft hat die folgenden Problemumgehungen getestet und gibt in der Beschreibung an, ob eine Problembehebung die Funktionalität einschränkt:
  
-   **Verwenden webbasierter E-Mail, wenn möglich**
  
    Falls verfügbar, verwenden Sie webbasierte E-Mail statt IMAP oder POP3, um Ihre E-Mail zu abzurufen.
  
#### Häufig gestellte Fragen (FAQs) zu der Sicherheitsanfälligkeit in Outlook Express und Windows Mail bezüglich Ganzzahlüberlaufs – CVE-2010-0816
  
**Worin genau besteht diese Sicherheitsanfälligkeit?**    
Diese Sicherheitsanfälligkeit kann eine Remotecodeausführung ermöglichen. Nutzt ein Angreifer diese Sicherheitsanfälligkeit erfolgreich aus, kann er die vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer kann dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerrechten erstellen.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Die Sicherheitsanfälligkeit wird verursacht, wenn eine gemeinsam von Outlook Express und Windows Mail verwendete Bibliothek Netzwerkdaten unzureichend überprüft, bevor mit diesen Daten die notwendige Größe eines Puffers berechnet wird.
  
**Was ist Windows Mail?**    
Windows Mail (früher Outlook Express) ist ein Online-Kommunikationstool zur Verwendung in Windows.
  
**Was sind die Mail-Protokolle POP3 und IMAP?**    
Auf den POP3-Servern (Post Office Protocol 3) werden eingehende E-Mail-Nachrichten bereitgehalten, bis Sie Ihre E-Mail abrufen. In diesem Moment werden sie auf Ihren Computer übertragen. POP3 ist der gängigste Kontotyp für persönliche E-Mail. Nachrichten werden in der Regel vom Server gelöscht, wenn Sie Ihre E-Mail abrufen. IMAP4 (Internet Message Access Protocol 4) ist ein Protokoll zum Lesen von Mail und für den Zugriff auf öffentliche Ordner auf Remoteservern. Clients können sich bei einem E-Mail-Server anmelden und mit IMAP4 Nachrichtenheader herunterladen, um die Nachrichten dann einzeln zu lesen, während sie online sind.
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann die gleichen Benutzerrechte wie der angemeldete Benutzer erlangen. Wenn ein Benutzer mit administrativen Benutzerberechtigungen angemeldet ist, kann ein Angreifer vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer kann dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerrechten erstellen. Für Endbenutzer, deren Konten mit weniger Benutzerrechten konfiguriert sind, kann dies geringere Auswirkungen haben als für Benutzer, die mit administrativen Benutzerrechten arbeiten.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
Ein Angreifer kann diese Sicherheitsanfälligkeit ausnutzen, indem er einen schädlichen E-Mail-Server einrichtet und den Client dazu verleitet, eine Verbindung zu diesem Computer herzustellen. Er antwortet dann mit einer speziell gestalteten POP3- oder IMAP-Antwort, durch die bewirkt wird, dass der Client die Sicherheitsanfälligkeit auslöst. Alternativ kann ein Man-in-the-Middle bestimmte Serverantworten bearbeiten und bewirken, dass diese Sicherheitsanfälligkeit ausgelöst wird.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Desktopsysteme, mit denen E-Mail gespeichert und gelesen wird, sind besonders durch diese Sicherheitsanfälligkeit gefährdet.
  
**Was bewirkt das Update?**    
Das Update behebt diese Sicherheitsanfälligkeit, indem vom E-Mail-Server zurückgegebene Antworten vor der Verarbeitung richtig überprüft werden.
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Nein. Microsoft erhielt Informationen über diese Sicherheitsanfälligkeit durch verantwortungsvolle Offenlegung. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich bekannt war.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Auch gab es keine Codebeispiele für ein Angriffskonzept.
