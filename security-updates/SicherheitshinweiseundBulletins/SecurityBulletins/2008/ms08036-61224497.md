Security Bulletin

Microsoft Security Bulletin MS08-036 - Kritisch
===============================================

Sicherheitsanfälligkeiten in Pragmatic General Multicast (PGM) können Denial-of-Service ermöglichen (950762)
------------------------------------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 10. Juni 2008 | Aktualisiert: Mittwoch, 17. Februar 2010

**Version:** 1.1

### Allgemeine Informationen

#### Kurzzusammenfassung

Dieses Sicherheitsupdate behebt zwei vertraulich gemeldete Sicherheitsanfälligkeiten im PGM-Protokoll (Pragmatic General Multicast), die einen Denial-of-Service ermöglichen können, wenn ein betroffenes System fehlerhafte PGM-Pakete empfängt. Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das System eines Benutzers nicht mehr reagiert, sodass ein Neustart erforderlich ist, um die Funktionalität wiederherzustellen. Beachten Sie, dass eine Sicherheitsanfälligkeit vom Typ Denial-of-Service einem Angreifer keine Codeausführung oder Erhöhung von Benutzerberechtigungen ermöglicht, sondern dazu führt, dass das betroffene System keine Anforderungen mehr annimmt.

Dieses Sicherheitsupdate wird für alle unterstützten Editionen von Windows XP und Windows Server 2003 als Hoch eingestuft und für alle unterstützten Editionen von Windows Vista und Windows Server 2008 als Mittel. Weitere Informationen finden Sie im Unterabschnitt **Betroffene und nicht betroffene Software** in diesem Abschnitt.

Das Sicherheitsupdate behebt diese Sicherheitsanfälligkeiten, indem geändert wird, wie PGM fehlerhafte Pakete analysiert. Weitere Informationen zur Sicherheitsanfälligkeit finden Sie im Unterabschnitt „Häufig gestellte Fragen (FAQs)“ für den bestimmten Sicherheitsanfälligkeitseintrag unter dem nächsten Abschnitt **Informationen zu Sicherheitsanfälligkeiten**.

**Empfehlung.**  Microsoft empfiehlt Benutzern, dieses Sicherheitsupdate so schnell wie möglich zu installieren.

**Bekannte Probleme.** Im [Microsoft Knowledge Base-Artikel 950762](http://support.microsoft.com/kb/950762) werden die derzeit bekannten Probleme dokumentiert, die durch die Installation dieses Sicherheitsupdates auftreten können. Im Artikel werden auch Lösungen für diese Probleme empfohlen. Wenn die derzeit bekannten Probleme und empfohlenen Lösungen nur auf bestimmte Veröffentlichungen dieser Software zutreffen, werden in diesem Artikel Verknüpfungen zu weiteren Artikeln bereitgestellt.

Sicherheitsanfälligkeit in PGM bezüglich ungültiger Länge – CVE-2008-1440  
-------------------------------------------------------------------------
  
<span></span>
Es liegt eine Sicherheitsanfälligkeit bezüglich Denial-of-Service in Implementierungen des PGM-Protokolls (Pragmatic General Multicast) unter Microsoft Windows XP und Windows Server 2003 vor. Die Sicherheitsanfälligkeit wird durch eine fehlerhafte Überprüfung speziell gestaltetet PGM-Pakete verursacht. Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass der Computer nicht mehr reagiert, sodass ein Neustart erforderlich ist, um die Funktionalität wiederherzustellen.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2008-1440](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1440).
  
#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit in PGM bezüglich ungültiger Länge – CVE-2008-1440
  
Schadensbegrenzung bezieht sich auf eine Einstellung, häufige Konfiguration oder allgemeine empfohlene Vorgehensweise, die in einem Standardzustand existieren und den Schweregrad der Ausnutzung einer Sicherheitsanfälligkeit verringern können. Die folgenden schadensbegrenzenden Faktoren könnten hilfreich für Sie sein:
  
-   Auf Systemen, auf denen Windows XP und Windows Server 2003 ausgeführt wird, wird Pragmatic General Multicast (PGM) nur aktiviert, wenn Microsoft Message Queuing (MSMQ) 3.0 installiert ist. Der MSMQ-Dienst ist standardmäßig nicht installiert.
  
#### Problemumgehungen für die Sicherheitsanfälligkeit in PGM bezüglich ungültiger Länge – CVE-2008-1440
  
Für diese Sicherheitsanfälligkeit gibt es bislang keine Problemumgehungen.
  
#### Häufig gestellte Fragen zur Sicherheitsanfälligkeit in PGM bezüglich ungültiger Länge – CVE-2008-1440
  
**Worin genau besteht diese Sicherheitsanfälligkeit?**    
Es handelt sich bei dieser Sicherheitsanfälligkeit um einen Denial-of-Service-Angriff. Ein anonymer Remoteangreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das betroffene System nicht mehr reagiert, bis das System neu gestartet wird. Beachten Sie, dass eine Sicherheitsanfälligkeit vom Typ Denial-of-Service einem Angreifer keine Codeausführung oder Erhöhung von Benutzerberechtigungen ermöglicht, sondern dazu führt, dass das betroffene System keine Anforderungen mehr annimmt.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Der Analysecode des Protokolls überprüft das Optionslängenfeld eines PGM-Pakets nicht richtig. Wenn ein betroffenes System ein speziell gestaltetes PGM-Paket mit einem ungültigen Optionslängenfeld erhält, geht der Parser in eine unendliche Schleife und bewirkt schließlich, dass das System nicht mehr reagiert.
  
**Was ist Pragmatic General Multicast (PGM)? **  
PGM ist ein zuverlässiges und skalierbares Multicastprotokoll, mit dem Empfänger den Verlust von Daten erkennen und die erneute Übertragung verlorener Daten anfordern bzw. eine Anwendung über endgültig verlorene Daten informieren können. PGM ist ein Empfänger-basiertes Protokoll, d. h. der Empfänger stellt sicher, dass alle Daten empfangen werden und nimmt dem Absender so die Verantwortung ab. PGM ist für Anwendungen geeignet, die eine Übertragung von Multicastdaten von mehreren Quellen zu mehreren Empfängern ohne Vervielfältigung erfordern. PGM unterstützt nicht die Bestätigung der Übertragung und garantiert nicht die Anforderung von Paketen von mehreren Absendern. Weitere Informationen zu PGM finden Sie im folgenden [MSDN-Artikel](http://msdn.microsoft.com/en-us/library/ms740125(vs.85).aspx).
  
**Was ist Microsoft Message Queuing?**  
Microsoft Message Queuing (MSMQ) ist eine Komponente des Windows-Betriebssystems, mit dem zusammenarbeitende Anwendungen sich gegenseitig Nachrichten senden und voneinander empfangen können. Dies gilt sogar, wenn die gewünschte Empfängeranwendung nicht ausgeführt wird oder der Computer, auf dem die Sender- bzw. Empfängeranwendung ausgeführt wird, abgeschaltet oder vom Netzwerk getrennt ist. Nachrichten werden gespeichert und von MSMQ weitergeleitet, bis sie die Zielwarteschlange erreichen. Später, wenn eine Empfängeranwendung ausgeführt wird, kann diese die Nachrichten aus der Warteschlange abrufen. MSMQ entkoppelt Sender- und Empfängeranwendungen, sodass diese nicht gleichzeitig ausgeführt werden müssen. Message Queuing stellt integrierte erhöhte Sicherheit, Transaktionsunterstützung und weitere Funktionen bereit. Weitere Informationen zu MSMQ finden Sie in [MSMQ: Häufig gestellte Fragen (FAQs)](http://www.microsoft.com/windowsserver2003/techinfo/overview/msmqfaq.mspx).
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das System eines Benutzers nicht mehr reagiert, sodass ein Neustart erforderlich ist, um die Funktionalität wiederherzustellen.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
Ein Angreifer kann versuchen, die Sicherheitsanfälligkeit auszunutzen, indem er ein speziell gestaltetes PGM-Paket an ein betroffenes System sendet. Das Paket kann dann bewirken, dass das betroffene System nicht mehr reagiert, bis es neu gestartet wird.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Arbeitsstationen und Server, auf denen MSMQ installiert ist oder eine PGM-unterstützte Anwendung ausgeführt wird (die auf PGM-Pakete wartet), sind hauptsächlich gefährdet.
  
**Was bewirkt das Update?**    
Das Update behebt die Sicherheitsanfälligkeit, indem das Optionslängenfeld von PGM-Paketen richtig überprüft wird.
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Nein. Microsoft erhielt Informationen über diese Sicherheitsanfälligkeit durch verantwortungsvolle Offenlegung. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich bekannt war.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Auch gab es keine Codebeispiele für ein Angriffskonzept.
  
Sicherheitsanfälligkeit in PGM bezüglich fehlerhafter Fragmente – CVE-2008-1441  
-------------------------------------------------------------------------------
  
<span></span>
Es liegt eine Sicherheitsanfälligkeit bezüglich Denial-of-Service in Implementierungen des PGM-Protokolls (Pragmatic General Multicast) unter Microsoft Windows XP und Windows Server 2003, Windows Vista und Windows Server 2008 vor. Der Analysecode des Protokolls überprüft speziell gestaltete PGM-Fragmente nicht richtig und bewirkt, dass das betroffene System nicht mehr reagiert, bis der Angriff endet.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2008-1441](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1441).
  
#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit in PGM bezüglich fehlerhafter Fragmente – CVE-2008-1441
  
Schadensbegrenzung bezieht sich auf eine Einstellung, häufige Konfiguration oder allgemeine empfohlene Vorgehensweise, die in einem Standardzustand existieren und den Schweregrad der Ausnutzung einer Sicherheitsanfälligkeit verringern können. Die folgenden schadensbegrenzenden Faktoren könnten hilfreich für Sie sein:
  
-   Auf Systemen, auf denen Windows XP oder Windows Server 2003 ausgeführt wird, wird Pragmatic General Multicast (PGM) nur aktiviert, wenn Microsoft Message Queuing (MSMQ) 3.0 installiert ist. Der MSMQ-Dienst ist standardmäßig nicht installiert.  
-   Auf Systemen, auf denen Windows Vista oder Windows Server 2008 ausgeführt wird, wird Pragmatic General Multicast (PGM) nur aktiviert, wenn Microsoft Message Queuing (MSMQ) 4.0 installiert und PGM speziell aktiviert ist. Der MSMQ-Dienst ist standardmäßig nicht installiert. Wenn der MSMQ-Dienst installiert ist, ist das PGM-Protokoll verfügbar, aber nicht standardmäßig aktiviert.
  
#### Problemumgehungen für die Sicherheitsanfälligkeit in PGM bezüglich fehlerhafter Fragmente – CVE-2008-1441
  
Für diese Sicherheitsanfälligkeit gibt es bislang keine Problemumgehungen.

Es handelt sich bei dieser Sicherheitsanfälligkeit um einen Denial-of-Service-Angriff. Ein anonymer Remoteangreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das betroffene System nicht mehr reagiert. Beachten Sie, dass eine Sicherheitsanfälligkeit vom Typ Denial-of-Service einem Angreifer keine Codeausführung oder Erhöhung von Benutzerberechtigungen ermöglicht, sondern dazu führt, dass das betroffene System keine Anforderungen mehr annimmt.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Der Analysecode des Protokolls verarbeitet fehlerhafte PGM-Pakete, die eine ungültige Fragmentoption enthalten, nicht richtig.
  
**Was ist Pragmatic General Multicast (PGM)? **  
PGM ist ein zuverlässiges und skalierbares Multicastprotokoll, mit dem Empfänger den Verlust von Daten erkennen und die erneute Übertragung verlorener Daten anfordern bzw. eine Anwendung über endgültig verlorene Daten informieren können. PGM ist ein Empfänger-basiertes Protokoll, d. h. der Empfänger stellt sicher, dass alle Daten empfangen werden und nimmt dem Absender so die Verantwortung ab. PGM ist für Anwendungen geeignet, die eine Übertragung von Multicastdaten von mehreren Quellen zu mehreren Empfängern ohne Vervielfältigung erfordern. PGM unterstützt nicht die Bestätigung der Übertragung und garantiert nicht die Anforderung von Paketen von mehreren Absendern. Weitere Informationen zu PGM finden Sie im folgenden [MSDN-Artikel](http://msdn.microsoft.com/en-us/library/ms740125(vs.85).aspx).
  
**Was ist Microsoft Message Queuing?**  
Microsoft Message Queuing (MSMQ) ist eine Komponente des Windows-Betriebssystems, mit dem zusammenarbeitende Anwendungen sich gegenseitig Nachrichten senden und voneinander empfangen können. Dies gilt sogar, wenn die gewünschte Empfängeranwendung nicht ausgeführt wird oder der Computer, auf dem die Sender- bzw. Empfängeranwendung ausgeführt wird, abgeschaltet oder vom Netzwerk getrennt ist. Nachrichten werden gespeichert und von MSMQ weitergeleitet, bis sie die Zielwarteschlange erreichen. Später, wenn eine Empfängeranwendung ausgeführt wird, kann diese die Nachrichten aus der Warteschlange abrufen. MSMQ entkoppelt Sender- und Empfängeranwendungen, sodass diese nicht gleichzeitig ausgeführt werden müssen. Message Queuing stellt integrierte erhöhte Sicherheit, Transaktionsunterstützung und weitere Funktionen bereit. Weitere Informationen zu MSMQ finden Sie in [MSMQ: Häufig gestellte Fragen (FAQs)](http://www.microsoft.com/windowsserver2003/techinfo/overview/msmqfaq.mspx).
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das System eines Benutzers nicht mehr reagiert.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
Ein Angreifer kann versuchen, die Sicherheitsanfälligkeit auszunutzen, indem er speziell gestaltete PGM-Pakete an ein betroffenes System sendet. Die Pakete können dann bewirken, dass das betroffene System nicht mehr reagiert. Das System kehrt zu normaler Funktionalität zurück, wenn alle speziell gestalteten Pakete verarbeitet wurden oder das System neu gestartet wird. Nach dem Neustart hört das System wieder auf, zu reagieren, wenn der Angreifer weiterhin die speziell gestalteten PGM-Pakete sendet.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Arbeitsstationen und Server, auf denen MSMQ installiert ist oder eine PGM-unterstützte Anwendung ausgeführt wird (die auf PGM-Pakete wartet), sind hauptsächlich gefährdet.
  
**Was bewirkt das Update?**    
Das Update behebt die Sicherheitsanfälligkeit, indem die Fragmentoptionen von PGM-Paketen richtig überprüft werden.
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Nein. Microsoft erhielt Informationen über diese Sicherheitsanfälligkeit durch verantwortungsvolle Offenlegung. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich bekannt war.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit für Angriffe auf Benutzer ausgenutzt wurde. Auch gab es keine Codebeispiele für ein Angriffskonzept.
