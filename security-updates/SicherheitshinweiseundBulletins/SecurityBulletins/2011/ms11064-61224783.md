Security Bulletin

Microsoft Security Bulletin MS11-064 - Kritisch
===============================================

Sicherheitsanfälligkeiten im TCP/IP-Stapel können Denial of Service ermöglichen (2563894)
-----------------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 9. August 2011

**Version:** 1.0

### Allgemeine Informationen

#### Kurzzusammenfassung

Dieses Sicherheitsupdate behebt zwei vertraulich gemeldete Sicherheitsanfälligkeiten in Microsoft Windows. Die Sicherheitsanfälligkeiten können Denial of Service ermöglichen, wenn ein Angreifer eine Sequenz von speziell gestalteten ICMP-Nachrichten (Internet Control Message Protocol) an ein Zielsystem sendet oder einem Server, der Webinhalte übermittelt und auf dem die URL-basierte QoS-Funktion (Quality of Service) aktiviert ist, eine speziell gestaltete URL-Anforderung sendet.

Dieses Sicherheitsupdate wird für alle unterstützten Editionen von Windows Server 2008 und Windows Server 2008 R2 als Hoch eingestuft und für alle unterstützten Editionen von Windows Vista und Windows 7 als Mittel. Weitere Informationen finden Sie im Unterabschnitt **Betroffene und nicht betroffene Software** in diesem Abschnitt.

Das Sicherheitsupdate behebt die Sicherheitsanfälligkeiten, indem geändert wird, wie der Windows TCP/IP-Stapel ICMP-Nachrichten bearbeitet und URLs im Speicher verarbeitet, wenn URL-basierter QoS aktiviert ist. Weitere Informationen zu den Sicherheitsanfälligkeiten finden Sie im Unterabschnitt „Häufig gestellte Fragen (FAQs)“ für den bestimmten Sicherheitsanfälligkeitseintrag im nächsten Abschnitt **Informationen zu Sicherheitsanfälligkeiten**.

**Empfehlung.** Die Mehrheit der Benutzer hat die automatische Aktualisierung aktiviert und muss keine Maßnahmen ergreifen, da dieses Sicherheitsupdate automatisch heruntergeladen und installiert wird. Benutzer, die die automatische Aktualisierung nicht aktiviert haben, müssen auf Updates prüfen und dieses Update manuell installieren. Weitere Informationen zu bestimmten Konfigurationsoptionen bei der automatischen Aktualisierung finden Sie im [Microsoft Knowledge Base-Artikel 294871](http://support.microsoft.com/kb/294871/de).

Für Administratoren und für Installationen in Unternehmen bzw. für Endbenutzer, die dieses Sicherheitsupdate manuell installieren möchten, empfiehlt Microsoft, das Update so schnell wie möglich mit Updateverwaltungssoftware zu installieren bzw. mithilfe des Diensts [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) auf Updates zu prüfen.

**Bekannte Probleme.** Keine

Sicherheitsanfälligkeit in ICMP bezüglich Denial of Service – CVE-2011-1871  
---------------------------------------------------------------------------
  
<span></span>
Im Windows TCP/IP-Stapel liegt eine Sicherheitsanfälligkeit bezüglich Denial of Service vor, die dadurch verursacht wird, dass der TCP/IP-Stapel eine Sequenz speziell gestalteter ICMP-Nachrichten falsch verarbeitet. Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das Zielsystem nicht mehr reagiert und dann automatisch neu startet.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2011-1871](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1871).
  
#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit in ICMP bezüglich Denial of Service – CVE-2011-1871
  
Für diese Sicherheitsanfälligkeit gibt es noch keine schadensbegrenzenden Faktoren.
  
#### Problemumgehungen für die Sicherheitsanfälligkeit in ICMP bezüglich Denial of Service – CVE-2011-1871
  
Problemumgehung bezieht sich auf eine Einstellung oder Konfigurationsänderung, die die zugrunde liegende Sicherheitsanfälligkeit nicht behebt, sondern die bekannten Angriffsmethoden blockiert, bevor Sie das Update installieren. Microsoft hat die folgenden Problemumgehungen getestet und gibt in der Beschreibung an, ob eine Problembehebung die Funktionalität einschränkt:
  
-   **Blockieren von ICMP an der Firewall**
  
    Durch das Blockieren aller ICMP-Nachrichten an der Firewall oder am Router werden hinter Firewall oder Router liegende Systeme vor dieser Sicherheitsanfälligkeit geschützt. Microsoft empfiehlt das Blockieren der gesamten unerwünschten eingehenden Kommunikation aus dem Internet.
  
    **Auswirkung der Problemumgehung:** Diese Problemumgehung kann sich negativ auf die Leistung auswirken, indem TCP an der Optimierung der Netzwerkkommunikation gehindert wird. ICMP-Netzwerkpakete können die Fragmentierung von Paketen bei Routern unterbinden, die Netzwerke mit unterschiedlichen MTUs verbinden. Fragmentierung führt zu reduziertem TCP-Durchsatz und verstärkter Netzwerkauslastung.
  
#### Häufig gestellte Fragen (FAQs) zur Sicherheitsanfälligkeit in ICMP bezüglich Denial of Service – CVE-2011-1871
  
**Worin genau besteht diese Sicherheitsanfälligkeit?**    
Es handelt sich bei dieser Sicherheitsanfälligkeit um einen Denial-of-Service-Angriff.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Die Sicherheitsanfälligkeit wird dadurch verursacht, dass der Windows TCP/IP-Stapel eine Sequenz speziell gestalteter ICMP-Nachrichten falsch verarbeitet.
  
**Welche Komponente ist von der Sicherheitsanfälligkeit betroffen?**    
Tcpip.sys ist die von dieser Sicherheitsanfälligkeit betroffene Komponente.
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das Zielsystem nicht mehr reagiert und dann automatisch neu startet.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
Ein nicht authentifizierter Remoteangreifer kann diese Sicherheitsanfälligkeit ausnutzen, indem er ein Programm erstellt, um eine Sequenz speziell gestalteter ICMP-Nachrichten an ein Zielsystem zu senden.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Server und Arbeitsstationen sind von dieser Sicherheitsanfälligkeit betroffen.
  
**Was bewirkt das Update?**    
Das Update behebt die Sicherheitsanfälligkeit, indem geändert wird, wie der Windows TCP/IP-Stapel ICMP-Nachrichten bearbeitet.
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Nein. Microsoft hat Informationen zu dieser Sicherheitsanfälligkeit durch eine koordinierte Offenlegung der Sicherheitsanfälligkeit erhalten.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich für Angriffe auf Benutzer ausgenutzt wurde.
  
Sicherheitsanfälligkeit in TCP/IP QoS bezüglich Denial of Service – CVE-2011-1965  
---------------------------------------------------------------------------------
  
<span></span>
Im Windows TCP/IP-Stapel liegt eine Sicherheitsanfälligkeit bezüglich Denial of Service vor, die dadurch verursacht wird, dass der TCP/IP-Stapel URLs im Speicher falsch verarbeitet, wenn der URL-basierte QoS (Quality of Service) aktiviert ist. Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das Zielsystem nicht mehr reagiert und dann automatisch neu startet.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2011-1965](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1965).
  
#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit in TCP/IP QoS bezüglich Denial of Service – CVE-2011-1965
  
Schadensbegrenzung bezieht sich auf eine Einstellung, häufige Konfiguration oder allgemeine empfohlene Vorgehensweise, die in einem Standardzustand existieren und den Schweregrad der Ausnutzung einer Sicherheitsanfälligkeit verringern können. Die folgenden schadensbegrenzenden Faktoren könnten hilfreich für Sie sein:
  
-   Standardmäßig ist die URL-basierte QoS-Funktion nicht auf jedem Windows-Betriebssystem aktiviert. Benutzer müssen diese Funktion manuell installieren, um von dieser Sicherheitsanfälligkeit betroffen zu sein.
  
#### Problemumgehungen für die Sicherheitsanfälligkeit in TCP/IP QoS bezüglich Denial of Service – CVE-2011-1965
  
Problemumgehung bezieht sich auf eine Einstellung oder Konfigurationsänderung, die die zugrunde liegende Sicherheitsanfälligkeit nicht behebt, sondern die bekannten Angriffsmethoden blockiert, bevor Sie das Update installieren. Microsoft hat die folgenden Problemumgehungen getestet und gibt in der Beschreibung an, ob eine Problembehebung die Funktionalität einschränkt:
  
-   Deaktivieren der richtlinienbasierten QoS-Funktion. Weitere Informationen finden Sie im Microsoft TechNet-Artikel [Richtlinienbasierte QoS (Quality of Service)](http://technet.microsoft.com/de-de/library/dd919203(ws.10).aspx).
  
#### Häufig gestellte Fragen (FAQs) zur Sicherheitsanfälligkeit in TCP/IP QoS bezüglich Denial of Service – CVE-2011-1965
  
**Worin genau besteht diese Sicherheitsanfälligkeit?**    
Es handelt sich bei dieser Sicherheitsanfälligkeit um einen Denial-of-Service-Angriff.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Die Sicherheitsanfälligkeit wird dadurch verursacht, dass der TCP/IP-Stapel speziell gestaltete URLs im Speicher falsch verarbeitet. Diese Sicherheitsanfälligkeit wird nur ausgelöst, wenn das Zielsystem Webinhalte mit einem Dienst wie IIS hostet und die URL-basierte QoS-Funktion aktiviert ist.
  
**Welche Komponente ist von der Sicherheitsanfälligkeit betroffen?**    
Tcpip.sys ist die von dieser Sicherheitsanfälligkeit betroffene Komponente.
  
**Was ist die URL-basierte QoS-Funktion (Quality of Service)?**    
QoS (Quality of Service) für Netzwerke ist ein branchenweiter Satz von Standards und Mechanismen zum Sicherstellen qualitativ hochwertiger Leistung für kritische Anwendungen. Netzwerkadministratoren können QoS-Mechanismen verwenden, um Prioritäten zu setzen und die Senderate für den ausgehenden Netzwerkverkehr zu verwalten. Durch die Verwendung dieser Mechanismen wird sichergestellt, dass Ressourcen effizient verwendet werden, um die erforderliche Qualitätsstufe bereitzustellen.
  
Mit Windows Server 2008 R2 können IT-Experten auf Basis der URL Prioritäten für den Internetdatenverkehr setzen. Mit der URL-basierter QoS-Funktion können IT-Experten sicherstellen, dass wichtiger Internetdatenverkehr vor weniger wichtigem Datenverkehr bearbeitet wird, wodurch die Leistung in verkehrsreichen Netzwerken verbessert wird. Beispielsweise können IT-Experten den Internetdatenverkehr von entscheidenden internen Websites eine höhere Priorität zuweisen als externen Websites. Dadurch wird die Leistung maximiert, wenn das Netzwerk ausgelastet ist. Weitere Informationen finden Sie im Microsoft TechNet-Artikel [URL-basierte QoS](http://technet.microsoft.com/de-de/library/dd637810(ws.10).aspx).
  
**Was ist eine URI?**    
Ein Uniform Resource Identifier (URI) ist eine Zeichenfolge, mit der Ressourcen im Internet oder über ein Netzwerk identifiziert werden bzw. auf diese eingewirkt wird. Eine URL ist ein typisches Beispiel für eine URI, die auf eine Ressource wie eine Website verweist. Weitere Informationen zu URIs finden Sie unter [RFC 2396](http://www.ietf.org/rfc/rfc2396.txt).
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das Zielsystem nicht mehr reagiert und dann automatisch neu startet.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
In einem Remoteangriffsszenario kann ein nicht authentifizierter Angreifer diese Sicherheitsanfälligkeit ausnutzen, indem er eine speziell gestaltete URL-Anforderung an einen Server sendet, der Webinhalte übermittelt und auf dem die URL-basierte QoS-Funktion aktiviert ist.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Server, die Webinhalte hosten und auf denen die URL-basierte QoS-Funktion aktiviert ist, sind hauptsächlich gefährdet. Server, auf denen die URL-basierte QoS-Funktion nicht aktiviert ist oder die keine Webinhalt übermitteln, sind nicht betroffen. Arbeitsstationen wie Windows 7-Systeme, die Internet Information Services (IIS) ausführen können, sind besonders gefährdet, wenn sowohl IIS als auch die URL-basierte QoS-Funktion aktiviert sind.
  
**Sind Windows 7-Systeme stärker gefährdet, wenn sie als Server mit IIS konfiguriert sind?**    
Ja. Serversoftware wie IIS kann auf Windows 7-Systemen installiert und konfiguriert werden. Obwohl Windows 7-Systeme als Arbeitsstationen betrachtet werden, agieren sie in dieser Konfiguration wie Server und sind deshalb stärker durch diese Sicherheitsanfälligkeit gefährdet.
  
**Was bewirkt das Update?**    
Das Update behebt die Sicherheitsanfälligkeit, indem geändert wird, wie der TCP/IP-Stapel URLs im Speicher verarbeitet, wenn die URL-basierte QoS-Funktion aktiviert ist.
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Nein. Microsoft hat Informationen zu dieser Sicherheitsanfälligkeit durch eine koordinierte Offenlegung der Sicherheitsanfälligkeit erhalten.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich für Angriffe auf Benutzer ausgenutzt wurde.
