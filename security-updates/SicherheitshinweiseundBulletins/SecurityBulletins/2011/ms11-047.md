Security Bulletin

Microsoft Security Bulletin MS11-047 - Kritisch
===============================================

Sicherheitsanfälligkeit in Hyper-V kann Denial-of-Service ermöglichen (2525835)
-------------------------------------------------------------------------------

Veröffentlicht: Dienstag, 14. Juni 2011

**Version:** 1.0

### Allgemeine Informationen

#### Kurzzusammenfassung

Dieses Sicherheitsupdate behebt eine vertraulich gemeldete Sicherheitsanfälligkeit in Windows Server 2008 Hyper-V und Windows Server 2008 R2 Hyper-V. Die Sicherheitsanfälligkeit kann Denial-of-Service ermöglichen, wenn ein authentifizierter Benutzer eine fehlerhafte Sequenz von Computeranweisungen auf einem der virtuellen Gastcomputer ausführt, die vom Hyper-V-Server gehostet wird. Ein Angreifer muss über gültige Anmeldeinformationen verfügen und speziell gestaltete Inhalte von einem virtuellen Gastcomputer senden können, um diese Sicherheitsanfälligkeit auszunutzen. Die Sicherheitsanfälligkeit kann nicht per Remotezugriff oder von anonymen Benutzern ausgenutzt werden.

Dieses Sicherheitsupdate wird für alle unterstützten x64-basierten Editionen von Windows Server 2008 und Windows Server 2008 R2 als Hoch eingestuft. Weitere Informationen finden Sie in dem Unterabschnitt **Betroffene und nicht betroffene Software** in diesem Abschnitt.

Das Sicherheitsupdate behebt die Sicherheitsanfälligkeit, indem korrigiert wird, wie der Hyper-V Server ungültige Anforderungen von virtuellen Computern bearbeitet. Weitere Informationen zur Sicherheitsanfälligkeit finden Sie im Unterabschnitt „Häufig gestellte Fragen (FAQs)“ für den bestimmten Sicherheitsanfälligkeitseintrag unter dem nächsten Abschnitt **Informationen zu Sicherheitsanfälligkeiten**.

**Empfehlung.** Die Mehrheit der Benutzer hat die automatische Aktualisierung aktiviert und muss keine Maßnahmen ergreifen, da dieses Sicherheitsupdate automatisch heruntergeladen und installiert wird. Benutzer, die die automatische Aktualisierung nicht aktiviert haben, müssen auf Updates prüfen und dieses Update manuell installieren. Weitere Informationen zu bestimmten Konfigurationsoptionen bei der automatischen Aktualisierung finden Sie im [Microsoft Knowledge Base-Artikel 294871](http://support.microsoft.com/kb/294871/de).

Für Administratoren und für Installationen in Unternehmen bzw. für Endbenutzer, die dieses Sicherheitsupdate manuell installieren möchten, empfiehlt Microsoft, das Update so schnell wie möglich mit Updateverwaltungssoftware zu installieren bzw. mithilfe des Diensts [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747&displaylang=de) auf Updates zu prüfen.

Siehe auch den Abschnitt **Tools und Anleitungen zur Erkennung und Bereitstellung** weiter unten in diesem Bulletin.

**Bekannte Probleme.** Keine

Häufig gestellte Fragen (FAQs) im Zusammenhang mit diesem Sicherheitsupdate
---------------------------------------------------------------------------

**Ich habe Hyper-V nicht aktiviert. Kann ich dieses Update installieren?**  
Nein. Das Update kann nur auf betroffenen Betriebssystemen installiert werden, auf denen die Hyper-V-Rolle aktiviert ist. Systeme, auf denen die Hyper-V-Rolle nicht aktiviert ist, sind nicht von der Sicherheitsanfälligkeit betroffen.

**Wird dieses Update angeboten, selbst wenn ich Hyper-V nicht aktiviert habe?**  
Das Update wird nur dann über Automatische Updates angeboten, wenn die Hyper-V-Rolle aktiviert ist. Dabei spielt es keine Rolle, ob das betroffene Betriebssystem mit der Server Core-Installationsoption installiert wurde oder nicht.

**Muss dieses Update auf virtuellen Gastcomputern von Hyper-V installiert werden?**  
Dieses Update muss nicht auf Gastcomputern von Hyper-V installiert werden, nur auf dem Computer, auf dem die Hyper-V-Rolle aktiviert ist. Es wird nur dem Hyper-V-Serversystem über Automatische Updates angeboten, nicht innerhalb der virtualisierten Gastcomputer.

**Was ist Hyper-V Server, und ist dieser ebenfalls von dieser Sicherheitsanfälligkeit betroffen?**  
Microsoft Hyper-V Server ist ein eigenständiges Produkt, mit dem eine zuverlässige und optimierte Virtualisierungslösung bereitgestellt wird, die Organisationen ermöglicht, die Servernutzung zu verbessern und Kosten zu verringern. Da Hyper-V Server ein dediziertes eigenständiges Produkt ist, das nur Windows Hypervisor-, Windows Server-Treibermodell- und Virtualisierungskomponenten enthält, stellt es geringen Platzbedarf und minimalen Aufwand bereit.

Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 und Microsoft Hyper-V Server 2008 R2 sind von dieser Sicherheitsanfälligkeit betroffen. Die jeweiligen Windows Server 2008- und Windows Server 2008 R2-Updates müssen installiert werden, um gegen diese Sicherheitsanfälligkeit geschützt zu sein. Benutzern, die Automatische Updates aktiviert haben, werden diese Updates automatisch zur Installation angeboten.

Sicherheitsanfälligkeit im VMBus bezüglich beständigen Denial-of-Services – CVE-2011-1872  
-----------------------------------------------------------------------------------------
  
In Hyper-V unter Windows Server 2008 und Windows Server 2008 R2 liegt eine Sicherheitsanfälligkeit bezüglich Denial-of-Services vor. Die Sicherheitsanfälligkeit besteht darin, dass Hyper-V Server bestimmte Reihenfolgen von Computeranweisungen unzureichend überprüft. Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das Hyper-V System nicht mehr reagiert. Dies würde sich auf alle virtuellen Computer auswirken, die von jenem System gehostet werden.
  
Informationen zum Anzeigen dieser Sicherheitsanfälligkeit als Standardeintrag in der Liste allgemeiner Sicherheitsanfälligkeiten finden Sie unter [CVE-2011-1872](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1872).
  
#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit im VMBus bezüglich beständigen Denial-of-Services – CVE-2011-1872
  
Schadensbegrenzung bezieht sich auf eine Einstellung, häufige Konfiguration oder allgemeine empfohlene Vorgehensweise, die in einem Standardzustand existieren und den Schweregrad der Ausnutzung einer Sicherheitsanfälligkeit verringern können. Die folgenden schadensbegrenzenden Faktoren könnten hilfreich für Sie sein:
  
-   Ein Angreifer benötigt gültige Anmeldeinformationen und muss sich lokal anmelden können, um diese Sicherheitsanfälligkeit auszunutzen. Die Sicherheitsanfälligkeit kann nicht per Remotezugriff oder von anonymen Benutzern ausgenutzt werden.
  
#### Problemumgehungen für die Sicherheitsanfälligkeit im VMBus bezüglich beständigen Denial-of-Services – CVE-2011-1872
  
Für diese Sicherheitsanfälligkeit gibt es bislang keine Problemumgehungen.
  
#### Häufig gestellte Fragen (FAQs) zur Sicherheitsanfälligkeit im VMBus bezüglich beständigen Denial-of-Services – CVE-2011-1872
  
**Worin genau besteht diese Sicherheitsanfälligkeit?**    
Es handelt sich bei dieser Sicherheitsanfälligkeit um einen Denial-of-Service-Angriff. Ein Angreifer, der diese Sicherheitsanfälligkeit ausnutzt, kann bewirken, dass der betroffene Hyper-V Server nicht mehr reagiert, wodurch ein Neustart erforderlich wird. Beachten Sie, dass eine Sicherheitsanfälligkeit vom Typ Denial-of-Service einem Angreifer keine Codeausführung oder Erhöhung von Benutzerberechtigungen auf dem Hyper-V Server ermöglicht, sondern dazu führt, dass der betroffene Dienst keine Anforderungen mehr annimmt.
  
**Was ist die Ursache dieser Sicherheitsanfälligkeit?**    
Die Sicherheitsanfälligkeit besteht aufgrund unzureichender Überprüfung bestimmter Pakete, die vom Gast an den Host gesendet werden. Aufgrund dieses Mangels bei der Überprüfung verursacht das Bearbeiten dieser Anwendungen eine überschüssige Nutzung von Ressourcen in der Hyper-V Serveranwendung.
  
**Was ist Hyper-V?**    
Hyper-V ist eine auf Hypervisor basierende Technologie, die eine Schlüsselfunktion von Windows Server 2008 und Windows Server 2008 R2 ist. Sie stellt eine skalierbare, zuverlässige und hochverfügbare Virtualisierungsplattform bereit. Diese Art von Umgebung ist nützlich, weil Sie virtuelle Computer erstellen und verwalten können. Dadurch können Sie mehrere Betriebssysteme auf einem physischen Computer ausführen und die Betriebssysteme voneinander isolieren.
  
**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**    
Ein Angreifer, der diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann bewirken, dass das betroffene System nicht mehr reagiert und neu gestartet werden muss. Beachten Sie, dass diese Sicherheitsanfälligkeit zu vollständiger Ressourcennutzung des Hyper-V Servers führen kann, einschließlich aller virtuellen Gastcomputer, die von jenem Server gehostet werden.
  
**Wie gehen Angreifer vor, um diese Sicherheitsanfälligkeit auszunutzen?**    
Ein Angreifer muss ein authentifizierter Administrator auf einem der virtuellen Gastcomputer sein, die vom Hyper-V Server gehostet werden. Der Angreifer muss die Möglichkeit haben, beliebigen Code auf jenem System auszuführen, z. B. durch Ausführen einer nicht vertrauenswürdigen ausführbaren Datei. Diese ausführbare Datei kann dann eine fehlerhafte Reihenfolge von Computeranweisungen aufrufen. Dies wiederum führt zu einer Ressourcennutzung in einer unendlichen Schleife auf dem Hyper-V Server. Dadurch wird das Verarbeiten von Anforderungen von anderen virtuellen Computern verhindert.
  
**Über welche Berechtigungen muss ein Angreifer verfügen, um diesen Denial-of-Service-Angriff auszuführen?**    
Um diesen Denial-of-Service-Angriff gegen den Hyper-V Server und alle virtuellen Computer durchzuführen, die auf dem Hyper-V Server ausgeführt werden, muss der Angreifer auf einem der virtuellen Computer, die auf dem Hyper-V Server gehostet werden, über Administratorrechte verfügen. Der Angreifer muss keinen Administratorzugriff auf den Hyper-V Server selbst haben.
  
**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**    
Hyper-V Server, die virtuelle Gastcomputer hosten, die nicht vertrauenswürdigen Benutzern Zugriff ermöglichen, sind besonders durch diese Sicherheitsanfälligkeit gefährdet.
  
**Was bewirkt das Update?**    
Durch dieses Update wird korrigiert, wie der Hyper-V Server ungültige Anforderungen von virtuellen Computern bearbeitet. Die Problembehebung ermöglicht der Hyper-V Serveranwendung, VMBus-Pakete richtig zu überprüfen, die vom Gast an den Host gesendet werden.
  
**War diese Sicherheitsanfälligkeit zum Zeitpunkt der Veröffentlichung dieses Security Bulletins bereits öffentlich bekannt?**    
Nein. Microsoft hat Informationen zu dieser Sicherheitsanfälligkeit durch eine koordinierte Offenlegung der Sicherheitsanfälligkeit erhalten.
  
**Lagen Microsoft zum Zeitpunkt der Veröffentlichung dieses Security Bulletins Informationen vor, dass diese Sicherheitsanfälligkeit bereits ausgenutzt wurde?**    
Nein. Microsoft lagen zum Zeitpunkt der Erstveröffentlichung dieses Security Bulletins keine Informationen vor, dass diese Sicherheitsanfälligkeit öffentlich für Angriffe auf Benutzer ausgenutzt wurde.

**Updatekompatibilitätsbewertung und Anwendungskompatibilitäts-Toolkit**
  
Updates bearbeiten oft dieselben Dateien und Registrierungseinstellungen, die zum Ausführen Ihrer Anwendungen benötigt werden. Dies kann eine Inkompatibilität auslösen und die Bereitstellung von Sicherheitsupdates verzögern. Mit den Komponenten zur [Updatekompatibilitätsbewertung](http://technet.microsoft.com/de-de/library/cc766043(ws.10).aspx), die im [Anwendungskompatibilitäts-Toolkit](http://www.microsoft.com/downloads/details.aspx?displaylang=de&familyid=24da89e9-b581-47b0-b45e-492dd6da2971&amp;displaylang=en) enthalten sind, können Sie die Vereinbarkeit von Windows-Updates mit installierten Anwendungen testen und überprüfen.
  
Das Microsoft Application Compatibility Toolkit (ACT) enthält alle notwendigen Tools und Dokumentationen, um die Anwendungskompatibilität zu prüfen und eventuelle Probleme zu beheben, bevor Microsoft Windows Vista, ein Windows-Update, ein Microsoft-Sicherheitsupdate oder eine neue Version von Windows Internet Explorer in Ihrer Umgebung bereitgestellt wird.
