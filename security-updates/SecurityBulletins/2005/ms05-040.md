Kurzzusammenfassung
-------------------

**Kurzzusammenfassung:**

Dieses Update behebt eine neue, von einem Privatanwender entdeckte Sicherheitsanfälligkeit. Der TAPI-Dienst (Telephony Application Programming Interface) weist eine Sicherheitsanfälligkeit auf, die eine Codeausführung von Remotestandorten aus ermöglichen kann. Die Sicherheitsanfälligkeit wird im Abschnitt „Einzelheiten zu dieser Sicherheitsanfälligkeit“ dieses Bulletins dokumentiert.

Nutzt ein Angreifer diese Sicherheitsanfälligkeit erfolgreich aus, kann er die vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer könnte dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerberechtigungen erstellen.

Wir empfehlen Benutzern, dieses Sicherheitsupdate so schnell wie möglich zu installieren.

Einzelheiten zu dieser Sicherheitsanfälligkeit
----------------------------------------------

#### Sicherheitsanfälligkeit im Telefoniedienst – CAN-2005-0058

Eine Sicherheitsanfälligkeit in TAPI (Telephony Application Programming Interface) kann die Codeausführung von Remotestandorten aus ermöglichen. Wenn ein Angreifer diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann er die vollständige Kontrolle über das betroffene System erlangen.

#### Schadensbegrenzende Faktoren für die Sicherheitsanfälligkeit im Telefoniedienst – CAN-2005-0058:

-   Eine Codeausführung von Remotestandorten aus ist nur auf Windows 2000 Server- und Windows Server 2003-basierten Systemen möglich, auf denen die Telefonieserver-Funktion aktiviert ist. Weitere Informationen zu dieser Funktion finden Sie auf der folgenden [Website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/de/library/serverhelp/a3574189-fd2f-4e12-90d9-c3ecb65a8226.mspx). Auf Systemen unter Windows 2000 Server und Windows Server 2003, auf denen die Telefondienst-Funktion nicht konfiguriert ist, sowie unter Windows 2000 Professional und Windows XP handelt es sich hierbei um eine Sicherheitsanfälligkeit, die zu einer lokalen Erhöhung von Berechtigungen führen kann.
-   Unter Windows Server 2003 ist der Telefoniedienst auf authentifizierte Benutzerkonten beschränkt, selbst dann, wenn das System als Telefonieserver aktiviert ist. Anonyme Angriffe sind unter Windows Server 2003 nicht möglich.
-   Standardmäßig wird der Telefoniedienst unter Windows XP und Windows Server 2003 nicht ausgeführt. Bei Bedarf startet der TAPI-Client den Telefoniedienst jedoch ohne jeden Benutzereingriff. Der Dienst kann von einem Benutzerkonto ohne Berechtigungen gestartet werden, es sein denn, er wurde von einem Administrator auf **Deaktiviert** eingestellt. Systeme, in denen der Telefoniedienst deaktiviert wurde, sind für dieses Problem nicht anfällig.
-   Mit Hilfe bewährter Methoden für die Firewall und standardisierten Firewallkonfigurationen können Netzwerke vor Remoteangriffen von außerhalb des Unternehmens geschützt werden. Eine bewährte Methode besteht darin, für Systeme, die mit dem Internet verbunden sind, nur eine minimale Anzahl von Ports zu öffnen.

#### Problemumgehungen für die Sicherheitsanfälligkeit im Telefoniedienst – CAN-2005-0058:

Microsoft hat die folgenden Problemumgehungen getestet. Diese Problemumgehungen beheben nicht die zugrunde liegende Sicherheitsanfälligkeit, sondern blockieren nur die bekannten Angriffsmethoden. Wenn die Funktionalität durch eine Problemumgehung verringert wird, so wird diese Einschränkung im folgenden Abschnitt genannt.

-   **Deaktivieren des Telefoniedienstes.**  
    Das Deaktivieren des Telefoniedienstes trägt zum Schutz vor Angriffen bei, mit denen diese Sicherheitsanfälligkeit ausgenutzt wird. Gehen Sie wie folgt vor, um den Telefoniedienst zu deaktivieren:

    1.  Klicken Sie auf **Start** und anschließend auf **Systemsteuerung** .Oder klicken Sie auf **Start**, zeigen Sie auf **Einstellungen**, und klicken Sie dann auf **Systemsteuerung**.
    2.  Doppelklicken Sie auf **Verwaltung**.
    3.  Doppelklicken Sie auf **Dienste**.
    4.  Doppelklicken Sie auf **Telefonie**.
    5.  Klicken Sie in der Liste **Starttyp** auf **Deaktiviert**.
    6.  Klicken Sie auf **Beenden**, und klicken Sie dann auf **OK**.

    Sie können den Telefoniedienst auch deaktivieren, indem Sie den folgenden Befehl in die Befehlszeile eingeben:

    `sc stop tapisrv & sc config tapisrv start= disabled`

    **Auswirkung der Problemumgehung:** Wenn der Telefoniedienst deaktiviert ist, schlagen sämtliche davon abhängigen Dienste und Betriebssystemfunktionen fehl. Beispielsweise würden u. a. der Assistent für neue Verbindungen, RAS-Verbindungen, modemgestützter DFÜ-Netzwerkbetrieb und der Faxdienst nicht ausgeführt werden können. Daher wird diese Problemumgehung nur für Systeme empfohlen, auf denen dieses Sicherheitsupdate nicht installiert werden kann.

-   **Blockieren Sie Folgendes an der Firewall:**  
    -   **Die UDP-Ports 135, 137, 138 und 445 sowie die TCP-Ports 135, 139, 445 und 593.**
    -   **Den unerwünschten eingehenden Datenverkehr mit Ports &gt; 1024.**
    -   **Alle weiteren speziell konfigurierten RPC-Ports.**

    Diese Ports dienen dazu, eine Verbindung mit RPC zu initiieren. RPC kann für die Remotekommunikation mit dem Telefoniedienst unter Windows 2000 Server und Windows Server 2003 verwendet werden. Das Blockieren dieser Ports an der Firewall schützt Systeme hinter dieser Firewall vor Angriffen, die diese Sicherheitsanfälligkeit auszunutzen versuchen. Sie sollten außerdem sicherstellen, dass auch alle weiteren speziell konfigurierten RPC-Ports auf dem Remotesystem blockiert sind. Wir empfehlen das Blockieren der gesamten unerwünschten eingehenden Kommunikation aus dem Internet. So können Sie Angriffe verhindern, bei denen möglicherweise andere Ports verwendet werden. Weitere Informationen zu von RPC verwendeten Ports finden Sie auf dieser [Website](http://go.microsoft.com/fwlink/?linkid=21312).

-   **Verwenden Sie zum Schutz vor netzwerkbasierten Versuchen, diese Sicherheitsanfälligkeit auszunutzen, eine persönliche Firewall, beispielsweise die**[**Internetverbindungsfirewall**](http://www.microsoft.com/germany/athome/security/content/protect/windowsxp/firewall.aspx)**, die im Lieferumfang von Windows Server 2003 enthalten ist.**  
    Die Internetverbindungsfirewall unter Windows Server 2003 schützt standardmäßig Ihre Internetverbindung, indem sie unerwünscht eingehenden Datenverkehr blockiert. Microsoft empfiehlt das Blockieren der gesamten unerwünschten eingehenden Kommunikation aus dem Internet.

    Führen Sie die folgenden Schritte durch, um die Internetverbindungsfirewall mit dem Netzwerkinstallations-Assistenten zu aktivieren:

    1.  Klicken Sie auf **Start** und anschließend auf **Systemsteuerung**.
    2.  Klicken Sie in der Kategorienansicht (Standardansicht) auf **Netzwerk- und Internetverbindungen**, und klicken Sie dann auf **Heimnetzwerk bzw. kleines Büronetzwerk einrichten oder ändern**. Die Komponente Internetverbindungsfirewall wird aktiviert, wenn Sie im Netzwerkinstallations-Assistenten eine Konfiguration auswählen, die angibt, dass Ihr System direkt mit dem Internet verbunden ist.

    Führen Sie die folgenden Schritte durch, um die Internetverbindungsfirewall manuell für eine Verbindung zu konfigurieren:

    1.  Klicken Sie auf **Start** und anschließend auf **Systemsteuerung**.
    2.  Klicken Sie in der Kategorienansicht (Standardansicht) auf **Netzwerk- und Internetverbindungen** und dann auf **Netzwerkverbindungen**.
    3.  Klicken Sie mit der rechten Maustaste auf die Verbindung, für die Sie die Internetverbindungsfirewall aktivieren möchten, und klicken Sie dann auf **Eigenschaften**.
    4.  Klicken Sie auf die Registerkarte **Erweitert**.
    5.  Aktivieren Sie das Kontrollkästchen **Diesen Computer und das Netzwerk schützen, indem das Zugreifen auf diesen Computer vom Internet eingeschränkt oder verhindert wird**, und klicken Sie dann auf **OK**.

    **Hinweis:** Wenn Sie die Kommunikation einiger Programme und Dienste über die Firewall aktivieren möchten, klicken Sie auf der Registerkarte **Erweitert** auf **Einstellungen**, und wählen Sie dann die notwendigen Programme, Protokolle und Dienste aus.

-   **Zum Schutz vor netzwerkbasierten Versuchen, diese Sicherheitsanfälligkeit auszunutzen, sollten Sie erweiterte TCP/IP-Filter auf Systemen aktivieren, die diese Funktion unterstützen.**  
    Sie können erweiterte TCP/IP-Filter aktivieren, um den gesamten unerwünschten eingehenden Datenverkehr zu blockieren. Weitere Informationen zur Konfiguration von TCP/IP-Filtern finden Sie im [Microsoft Knowledge Base-Artikel 309798](http://support.microsoft.com/kb/309798).

-   **Zum Schutz vor netzwerkbasierten Versuchen, diese Sicherheitsanfälligkeit auszunutzen, sollten Sie die betroffenen Ports mit IPSec auf den betroffenen Systemen blockieren.**  
    Mit Hilfe von IPSec (Internet Protocol Security) können Sie die Netzwerkkommunikation sicherer gestalten. Ausführliche Informationen zu IPSec und dem Anwenden von Filtern finden Sie im [Microsoft Knowledge Base-Artikel 313190](http://support.microsoft.com/kb/313190) und im [Microsoft Knowledge Base-Artikel 813878](http://support.microsoft.com/kb/813878).

#### Häufig gestellte Fragen (FAQs) zur Sicherheitsanfälligkeit im Telefoniedienst – CAN-2005-0058:

**Worin genau besteht diese Sicherheitsanfälligkeit?**  

Diese Sicherheitsanfälligkeit kann je nach Betriebssystemversion und Konfiguration die Codeausführung von Remotestandorten aus oder die Erhöhung lokaler Berechtigungen ([Privilege Elevation](http://go.microsoft.com/fwlink/?linkid=21142)) ermöglichen. Weitere Informationen finden Sie im Abschnitt „Häufig gestellte Fragen“ unter der Frage „Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?“. Nutzt ein Angreifer diese Sicherheitsanfälligkeit erfolgreich aus, kann er die vollständige Kontrolle über ein betroffenes System erlangen. Ein Angreifer könnte dann Programme installieren, Daten anzeigen, ändern oder löschen oder neue Konten mit sämtlichen Benutzerberechtigungen erstellen.

**Was ist die Ursache dieser Sicherheitsanfälligkeit?**  
Diese Sicherheitsanfälligkeit ergibt sich durch den vom Telefoniedienst verwendeten Prozess zum Überprüfen von Daten und Berechtigungen.

**Was ist der Telefoniedienst?**  
Der Telefoniedienst stellt Unterstützung für TAPI (Telephony Application Programming Interface) bereit. TAPI ist für die Integration von Telekommunikation in das Betriebssystem zuständig. TAPI unterstützt sowohl herkömmliche als auch IP-Telefonie, um Sprach-, Daten- und Videokommunikation zu ermöglichen. Zu der unterstützten Hardware zählen Sound- und Videokarten, Modems, ISDN-Leitungen, ATM-Netzwerke und Kameras. Mithilfe dieser Hardware können Sie über direkte Verbindungen mit lokalen Computern, Telefonleitungen, LANs, WANs und dem Internet kommunizieren.
Neben dem Tätigen und Empfangen von Anrufen können Programme TAPI zur Bereitstellung besonderer Telefoniefunktionen, wie Anrufer-ID, Anrufweiterleitung, Voice-Mail und Videokonferenzen, verwenden. Kommunikationsprogramme können in der Lage sein, den Anrufer zu identifizieren, Anruferinformationen zu speichern und anzuzeigen oder sogar einem Anruf auf der Basis von Kundeninformationen eine Priorität zuzuweisen oder ihn weiterzuleiten. Weitere Informationen zu TAPI finden Sie auf der folgenden [Website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/serverhelp/7d644322-d216-484c-ab74-028ae65f2116.mspx).

**Was kann ein Angreifer über diese Sicherheitsanfälligkeit erreichen?**  
Wenn ein Angreifer diese Sicherheitsanfälligkeit erfolgreich ausnutzt, kann er die vollständige Kontrolle über das betroffene System erlangen.

**Wer könnte diese Sicherheitsanfälligkeit ausnutzen?**  
Jeder anonyme Benutzer, der eine speziell gestaltete Nachricht an das betroffene System übermitteln kann, könnte unter Windows 2000 Server versuchen, diese Sicherheitsanfälligkeit auszunutzen.
Unter Windows Server 2003 ist der Telefoniedienst auf authentifizierte Benutzerkonten beschränkt.
Unter Windows 2000 Professional und Windows XP führt diese Sicherheitsanfälligkeit zu einer Erhöhung der Berechtigungen. Um die Sicherheitsanfälligkeit auszunutzen, muss sich der Angreifer lokal bei einem System anmelden und ein Programm unter Windows 2000 Professional bzw. Windows XP ausführen können.

**Für welche Systeme stellt diese Sicherheitsanfälligkeit hauptsächlich ein Risiko dar?**  
In erster Linie sind als Telefonieserver konfigurierte Windows 2000-Server dem Risiko anonymer Angreifer ausgesetzt. Ein Administrator muss jedoch bestimmte Schritte ausführen, um einen Windows 2000 Server-Computer als Telefonieserver zu konfigurieren. Eine Beschreibung dieser Schritte finden Sie auf dieser [Website](http://www.microsoft.com/windows2000/en/advanced/help/default.asp?url=/windows2000/en/advanced/help/tapi_server_enable_disable.htm).
Windows Server 2003-Systeme, die als Telefonieserver konfiguriert wurden, sind durch authentifizierte Benutzer gefährdet. Ein Administrator muss jedoch bestimmte Schritte ausführen, um einen Windows Server 2003-Computer als Telefonieserver zu konfigurieren. Eine Beschreibung dieser Schritte finden Sie auf dieser [Website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/de/library/serverhelp/a3574189-fd2f-4e12-90d9-c3ecb65a8226.mspx).
Auf Systemen, die nicht als Telefonieserver eingerichtet sind, unter Windows 2000 Professional sowie unter Windows XP kann diese Sicherheitsanfälligkeit zu einer Erhöhung der lokalen Berechtigungen führen.
Systeme, auf denen der Telefoniedienst deaktiviert ist, sind von dieser Sicherheitsanfälligkeit nicht betroffen.

**Sind Windows 98, Windows 98 Second Edition oder Windows Millennium Edition auf kritische Weise von dieser Sicherheitsanfälligkeit betroffen?**  
Nein. Obwohl Windows 98, Windows 98 Second Edition und Windows Millennium Edition die betroffenen Komponenten enthalten, ist die Sicherheitsanfälligkeit nicht kritisch.
